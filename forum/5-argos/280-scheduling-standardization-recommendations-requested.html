Hi Evelyn,

 

Until recently, we allowed all datablock designers (currently 55) to create and modify schedules.  This ended up being a mess as some people were bypassing our review process.  Now, only our two administrators can create/modify schedules.  Requests to create/modify schedules are sent to a shared email box for reporting.  

 

We typically do not have errors enabled on our schedules.  Errors typically only happen when one of our data sources go down.  When this happens, we just re-run all the scheduled reports for that data source.  When errors are enabled, they go to the shared email box.  When users want the "Fail if report query returns 0 records) checkbox enabled, those errors will then go to the requester or a shared email box to which they have access.

 

Here are some of the standards we use:

 

  • Schedule names start with their frequency (e.g., "Daily", "Weekly", etc).  
  • Schedules can only run during business hours and only run on the hour or on the half hour.  Schedules are set to run at 7:00 AM by default unless there is a business need to have them sent later in the day.  
  • Emails are sent in HTML format.
  • Emails are sent by default from the shared reporting email address.  This way, users can reply directly to the email if they need.  Some departments (e.g., Bursar & HR) have reports coming from one of their shared email addresses.  In general, emails should not be sent from a person.
  • The email body includes a standard section on how the user can re-run that particular report (if they have access).  The body also includes a copy of the schedule path it make it easier for administrators to find.  
  • All of our schedules run as a separate "Scheduler" account.  This was necessary as not all datablock designers have access to data needed in the report.

Hope this helps!
