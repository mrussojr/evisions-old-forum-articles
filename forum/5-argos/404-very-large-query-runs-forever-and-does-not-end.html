Does it run long in Argos, or in your SQL editor (TOAD, Tora, SQL navigator, etc.)? If it's in both, try running an explain plan on the query inside the SQL editor and reducing the use of full table scans when possible:

 

explain plan for (
      ...paste your query here...
 );

;

 

I noticed you use a lot of tables here that at our institution contain millions of records - SHRLGPA, SORLFOS, SFRSTCR, TBRACCD, etc. - for those you could try adding an "where exists" clause to only get records for people currently enrolled/registered in the term. For example:

 

(select tbraccd_pidm term_pidm, sum(tbraccd_balance) term_bal
                     from tbraccd
                    where tbraccd_term_code = :parm_dd_term
                     and exists (select sfrstcr_pidm from sfrstcr where sfrstcr_term_code = :parm_dd_Term and sfrstcr_pidm = tbraccd_pidm and sfrstcr_rsts_code in ('RE','AU', 'RW'))
                   group by  tbraccd_pidm)

 

If it's JUST Argos causing the problem, you might try adding bits of the query slowly until you isolate the problem table.
