I do not believe you can limit directly.  However, you could add a row number to each item, and then select from that.  For example:

  

SELECT 
    EMPLOYEEID,
    LAST_NAME,
    FIRST_NAME,
    LISTAGG(SIMPLE_NAME, ';') WITHIN GROUP (ORDER BY SIMPLE_NAME) GroupList
FROM (    
    SELECT
        u.EMPLOYEEID,
        u.LAST_NAME,
        u.FIRST_NAME,
        g.SIMPLE_NAME,    
        ROW_NUMBER() OVER (PARTITION  BY u.EmployeeID ORDER BY g.GROUP_NAME) GroupRank         
    FROM
        AD_USER u
        JOIN AD_GROUP_MEMBERSHIP m ON (u.EMPLOYEEID = m.EMPLOYEEID)
        JOIN AD_GROUP g ON (m.GROUP_NAME = g.GROUP_NAME)
)
WHERE 
    GroupRank <= 3
GROUP BY 
    EMPLOYEEID,
    LAST_NAME,
    FIRST_NAME;

 

In this query, I am getting a list of groups for each user.  In the inner query, a sequential number is assigned to each group name and resets for each employee.  In the outer query, I limit the number of groups returned to the first three.

 

Hope that helps!

Oracle has a rownum field that you can use  - It may be possible to do something like:

 

select LISTAGG (MEMO_ID_ID , ';')

       WITHIN GROUP (ORDER BY MEMO_ID_ID)

       FROM SELECT (MEMO_ID_ID FROM BANINST1.AA_GIFT_AND_MEMO_DETAIL AA_GIFT_AND_MEMO_DETAIL

       WHERE AA_GIFT_AND_MEMO_DETAIL.PIDM_KEY = AV_ALL_CONSTITUENTS.PIDM_KEY)

      WHERE ROWNUM  < 4
