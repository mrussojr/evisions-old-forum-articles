In cases like that, I schedule the report to run every day but in the WHERE clause of the query I include a statement to the effect of 'trunc(sysdate) = trunc(date_being_checked)'.

 

Technically the report runs every day, but the query will generate results only when the dates match.

Similarly to Todd, we have dozens of reports scheduled to run every day, but they only send out emails based on the current data, so some days nothing is sent out.  Sometimes, that is looking at particular dates (such as our reminders to faculty to turn in grades) and sometimes it is based on the existence of data matching the criteria needed.  We have lots of scheduled reports that send emails based on the activity date in a Banner table being in the last day (e.g., sfrstcr_activity_date >= sysdate - 1).  I don't know of any way to dynamically change the scheduled run of a report schedule.
