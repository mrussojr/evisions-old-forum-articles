Answer:  According to our research and some thorough testing and collaborating with the evisions helpdesk, we have found this to be a network issue.  Specifically, we have a load balancer set up on our network (they call it the A10) which likes to randomly drop the connection.  Our specific solution was to use the exact server name in the maps url (not an alias) and users have to be logged into our virtual private network (somehow I'm told this avoids the load balancer).  That's our configuration and others may have something different.
