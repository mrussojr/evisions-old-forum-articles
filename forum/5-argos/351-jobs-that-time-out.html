We set up ALL our connections (Oracle, Access, SQLite, etc) using ADO connections.  We did this for two reasons.

 

  1. We previously had connections for our both Prod and Test databases set up in MAPS.  When we copied our Production MAPS over to our Test server it put a connection to the Prod database in Test, which is less then desirable.  By creating ADO connections, we had a single reference the ADO Connection on the server (called it Banner), which was pointed to Prod DB on Production server and Test DB on Test server.
  2. Uniformity.

Using this approach, we were able to overcome the connection timeouts by un-checking the 'Enable Query Timeout' checkbox on the Oracle ODBC Driver Configuration for that connection which solved all of our timeout issues.

 

Figuring out how to get the ADO connections working was a bit of a struggle (not hard, but there are a few hidden must-do's) but if you need any help, let me know.  I highly recommend this approach.
