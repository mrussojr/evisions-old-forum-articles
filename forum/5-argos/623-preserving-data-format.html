Good morning,

 

This is not a flaw in Argos as much as a design decision in how Excel opens some files.  I generally see this when people open .CSV files in Excel.  If you look at the raw file (say in Notepad++), the leading zeros are there.  But, once opened in Excel, they are converted to numbers.

 

One way to mitigate this is to wrap the field in double quotes and then precede it with an equal sign.  For example: 

SELECT '="' || SPRADDR_ZIP || '"'
FROM SPRADDR

(You could also save the file as .TXT and use the import wizard in Excel.  But, most users do not want to do that.)

HTH
