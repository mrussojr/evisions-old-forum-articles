So, I was able to put ~10K recipients in the To: column, but I didn't try actually sending the email.  I would recommend against this if practical, especially if mailing externally:  A number of mail server providers will reject emails that have more than 100 recipients (To, CC, and BCC combined).  Also, it can be a pain to manage such a list in a text box.  

 

If you can, I would see if you can create a distribution list.  Then, Argos only needs to send a mail with one recipient.  This also means that someone else is able to manage the distribution list.  Another option would be to create a SQL variable that pulls a distribution list.
