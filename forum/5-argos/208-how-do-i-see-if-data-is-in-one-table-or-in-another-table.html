Try this:

 

SELECT

TERM,

PIDM,

CASE

WHEN VETN_CODE IS NOT NULL AND DETAIL_CODE IS NOT NULL

THEN 'BOTH SOURCES'

WHEN VETN_CODE IS NOT NULL

THEN 'VETN'

WHEN DETAIL_CODE IS NOT NULL

THEN 'TBRACCD'

END AS DATA_SOURCE

FROM

( SELECT DISTINCT

PIDM,

TERM,

MAX (T_1) AS VETN_CODE,

MAX (T_2) AS DETAIL_CODE

FROM

(SELECT DISTINCT

SGRVETN_PIDM AS PIDM,

SGRVETN_TERM_CODE_VA AS TERM,

'SGRVETN' AS T_1,

NULL AS T_2

FROM

SATURN.SGRVETN

WHERE

SGRVETN_TERM_CODE_VA = &&TERM ----INSERT VARIABLE HERE

UNION

SELECT DISTINCT

TBRACCD_PIDM AS PIDM,

TBRACCD_TERM_CODE TERM,

NULL AS T_1,

'TBRACCD' AS T_2

FROM

TAISMGR.TBRACCD

WHERE

TBRACCD_TERM_CODE = &&TERM ----INSERT VARIABLE HERE

AND TBRACCD_DETAIL_CODE = 'GVBD' --- code I used to test with

--AND TBRACCD_DETAIL_CODE IN(YOURLIST)

)

GROUP BY

PIDM, TERM

)

Dave - two thoughts:

 

1. We have issues running TBRACCD queries as well because the table has so few (practical) indices and trillions (!) of records. Depending on what you are doing, you could add an additional index on that field if your DBA will allow, make a view with the additional index and query that instead, OR cut down what you return with an "exists" in your where. We often need data on students who are currently registered only, so adding an exists to SFRSTCR with certain rsts codes cuts the returned rows down tremendously.

 

2. Just as another option - we have a variety of veteran information stored in a variety of places. I used one table as my driver (SFRSTCR in my case) then left joined on everything else. Then I used a coalesce() function in my where clause, listing all the possible columns related to veteran data, when the column wasn't null, i.e.

 

.....

where coalesce(col1,col2,col3,col,col5...) is not null

 

Just my two cents.

I'm not sure I understand what you mean, but how about this:

 

If you want to just list the source as VETN if it exists in both places:

 

CASE

------WHEN VETN_CODE IS NOT NULL AND DETAIL_CODE IS NOT NULL

-------THEN 'BOTH SOURCES'

  WHEN VETN_CODE IS NOT NULL

    THEN 'VETN'

  WHEN DETAIL_CODE IS NOT NULL

    THEN 'TBRACCD'

END AS DATA_SOURCE

 

If you want to list the actual VETN_CODE as well as the source alter it or add a second case statement:

 

CASE

  WHEN VETN_CODE IS NOT NULL

    THEN VETN_CODE

  WHEN DETAIL_CODE IS NOT NULL

    THEN DETAIL_CODE

END AS DATA_CODE

 

The issue here might be that I used the MAX function to get an unduplicated result from the sources since multiple codes could exist in your data.  if you have a hierarchy of codes you could do a ranking and then a decode/case process to return the most useful code for your purpose.

I think I finally get it.  I wasn't seeing that there were multiple records being returned.  The code below should work much better:

 

SELECT DISTINCT

PIDM,

TERM,

CASE

  WHEN SGRVETN_PIDM IS NULL

    THEN 'TBRACCD DATA'

  ELSE 'SGRVETN DATA'

END DATA_SOURCE,

CASE

   WHEN SGRVETN_PIDM IS NULL

      THEN NULL

   ELSE STVVETC_DESC

END VETERAN_DESCRIPTION

FROM

SATURN.SGRVETN,

SATURN.STVVETC,

(

SELECT DISTINCT

SGRVETN_PIDM AS PIDM,

SGRVETN_TERM_CODE_VA AS TERM

FROM SATURN.SGRVETN

WHERE SGRVETN_TERM_CODE_VA = &&TERM

UNION

SELECT DISTINCT

TBRACCD_PIDM AS PIDM,

TBRACCD_TERM_CODE AS TERM

FROM TAISMGR.TBRACCD

WHERE

TBRACCD_TERM_CODE = &&TERM ----INSERT VARIABLE HERE

PIDM = SGRVETN_PIDM (+)
