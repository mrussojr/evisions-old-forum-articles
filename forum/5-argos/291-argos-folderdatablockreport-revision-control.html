In the coming weeks we will be implementing Revision Control for all folders, datablocks, and reports.  We are doing this to eliminate the numerous copies of datablocks that exist in various locations because users are afraid to delete old data (some are 5+ years old).  Some of these duplicate reports have schedules attached and while not active, it has completely cluttered up the Schedules section in MAPS.  I also consider Revision Control a best practice .

 

While running through some testing I've noticed that there isn't a great way to implement Revision Control site-wide, or not that I've found yet.  I was planning on setting it at the parent folder level (just below root to eliminate the Recycle Bin and our personal folder) and having all sub folders, datablocks, and reports set to 'Use Parent Settings (Revision Control = YES)'.  After setting the parent folder, I quickly realized that I would then have to set each sub folder individually to use the parent settings which isn't bad as there is only about 25.  But they each have sub-folders... and some of the sub-folders have sub-folders.  Then there's the issue of datablocks, hundreds of them, and reports.  Suddenly this didn't sound fun anymore.

 

I went digging through the ArgosData.sqlite database to see if I could determine where these settings are and if I could do a bulk update on all folders, datablocks and reports.  Here's what I've found.

 

It appears RevisionControl has the following values: null = No, 1 = Yes, 2 = Use Parent Settings

 

Folders:
The parent folder can be singled out and updated in the Folder table with 'set RevisionControl = 1, MaxRevisions = XX' where XX is the desired number of revision history.
All sub-folders of the parent can be updated in the Folder table with 'set RevisionControl = 2, MaxRevisions = null'

 

Datablocks:
All datablocks in a sub-folder or directly within the parent folder can be updated in the DataBlock table with 'set RevisionControl = 2, MaxRevisions = ??'
I say ?? because it appears that all records in the DataBlock table where RevisionControl is set to 2 default to 10, even though the MaxRevisions of the Parent Settings may differ.

 

Reports:
All reports belonging to a datablock in a sub-folder or directly within the parent folder can be updated in the Report table with 'set RevisionControl = 2, MaxRevisions = ??'
Again, ?? because it appears that all records in the Report table where RevisionControl is set to 2 default to 10, even though the MaxRevisions of the Parent Settings may differ.

 

Has anyone ever done this or is there a better way to implement Revision Control throughout Argos without doing it on every single folder/datablock/report?  Also, for those that have already implemented Revision Control, are there any best practices or tips and tricks you would like to share?
