Looks like it is looking for the parenthesis for the IF statement.  IF( {emp_class} IN ('AC', 'AH', 'AL', 'FA'), 'ADJ', 'TOAST).  Suarez has it right using the fn button catches those little things that we often forget.  You could also use decode instead of an IF statement, decode won't work with IN, but you can nest them.  decode({emp_class}, 'AC' , 'ADJ', decode({emp_class}, 'AH' , 'ADJ', decode({emp_class}, 'AL' , 'ADJ', decode({emp_class}, 'FA' , 'ADJ', 'TOAST'))))

So, it's settled that there's no IN or CASE statement in the expression builder, according to Evision.  Based on a suggestion, I finally wrote the following piece of art:

 

If({ArgosData.nbrjobs_ecls_code} = 'AC', 'ADJ', If({ArgosData.nbrjobs_ecls_code}= 'AD', 'ADM', If({ArgosData.nbrjobs_ecls_code}= 'AH', 'ADJ', If({ArgosData.nbrjobs_ecls_code}= 'AL', 'ADJ', If({ArgosData.nbrjobs_ecls_code}= 'CF', 'CMSU', If({ArgosData.nbrjobs_ecls_code}= 'CI', 'CLS', If({ArgosData.nbrjobs_ecls_code}= 'CL', 'CLS', If ({ArgosData.nbrjobs_ecls_code}= 'CS', 'CLS', If ({ArgosData.nbrjobs_ecls_code}= 'DS', 'CMSU', If ({ArgosData.nbrjobs_ecls_code}= 'F0', 'FAC', If ({ArgosData.nbrjobs_ecls_code}= 'F1', 'FAC', If ({ArgosData.nbrjobs_ecls_code}= 'F2', 'FAC', If ({ArgosData.nbrjobs_ecls_code}= 'FA', 'ADJ', If ({ArgosData.nbrjobs_ecls_code}= 'FC', 'FAC', If ({ArgosData.nbrjobs_ecls_code}= 'FL', 'FAC', If ({ArgosData.nbrjobs_ecls_code}= 'FV', 'FAC OVL', If ({ArgosData.nbrjobs_ecls_code}= 'FX', 'FAC OVL', If ({ArgosData.nbrjobs_ecls_code}= 'MG', 'CMSU', If ({ArgosData.nbrjobs_ecls_code}= 'PI', 'HRLY', If ({ArgosData.nbrjobs_ecls_code}= 'PX', 'HRLY', If ({ArgosData.nbrjobs_ecls_code}= 'SH', 'HRLY', If ({ArgosData.nbrjobs_ecls_code}= 'FB', 'FAC', If ({ArgosData.nbrjobs_ecls_code}= 'SI', 'STW',If ({ArgosData.nbrjobs_ecls_code}= 'SP', 'CMSU', If ({ArgosData.nbrjobs_ecls_code}= 'SR', 'ADM', If ({ArgosData.nbrjobs_ecls_code}= 'ST', 'STW', If ({ArgosData.nbrjobs_ecls_code}= 'TR', 'BRD','OTHER')))))))))))))))))))))))))))
