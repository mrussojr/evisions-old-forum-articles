Hi David,

You can add fields from any dataset (including drop downs and multi-columns) in your block to reports that you create as banded or extract. There is no way to directly link a form and create a report. You'll need to know what all of your datasets and fields are called so you can select exactly what to bring into the report.

I recreated your report query and created variables like you did.  The first thing I noticed are the lines:  AND :Parm_BT_Run  is not null

 

and when I ran the datablock and clicked the Get Report button, my browser threw an error:

 

Error creating the report: Error result code: 1

This report cannot be run due to the following errors: Could not find variable "PARM_BT_RUN" Could not find variable "PARM_BT_RUN" 

Check the server log for more information.

 

So the first thing to do is to remove those lines:  AND :Parm_BT_Run  is not null

You don't need these lines in a report query since it won't run anyways until it is invoked.

 

Second, there is an issue if more than one value is selected in the listboxes.  It looks like it can't configure the URL properly based on my variable example.  So for now, until I figure out how to configure the RptUrl value to allow for multiple selected values, it's best to use a drop down to avoid user error.
