We did what Justin suggested.  The API on the report was enabled and the OnClick event for the button was hyperlink.  The hyperlink value is an expression, so we can create an expression that uses both the api url and the variable expressions that the report query needs in order to run.  Here's an example of the value we used for the hyperlink event behind the button:

 

https://maps.rollins.edu/mrr?report=XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX&D_mtcReportList.RptID=%%:D_mtcReportList.RptID%%&varUsername=%%:varUsername.CurrentUser%%

We use multi-column boxes to display the reports, so the person makes the selection of the parameters, then they a search button, and then the report displays on the multi-column box on the same dashboard. Then they can simply right click on the multi-column box to export. I have attached an image of one of the dashboards to give you an idea. So when they click the search button, the list populates in the Quick view and then they can click the gear on the top right corner to export to excel. 

 

I am not sure if this helps, but this way, the user does not have to run the report from the top.
