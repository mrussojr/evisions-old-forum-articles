Tab ordering is set by the Tab Order property.  How it generally works is that it starts at the lowest Tab Order for the form and goes upward.  If multiple objects have the same Tab Order, that may cause issues.  Also, since you can nest controls (e.g., have buttons inside of panels), Tab Order values do not have to be unique for the entire form.  They only need to be unique at the nesting level.  Say for example you have a form with three buttons and two of those buttons are in a panel, here is how I may set up the Tab Order.

  • Button #1: Tab Order 0
  • Panel:  Tab Order 1
    • Button #2: Tab Order 0
    • Button #3: Tab Order 2

If you already have a complex form built, you can set the Tab Order relatively easily by starting with the last control to receive focus, set its Tab Order to zero, move to the second-to-last control, set its Tab Order to zero, ... , until you get to the first control, and set its Tab Order to zero.  Argos will set ordering as appropriate.
