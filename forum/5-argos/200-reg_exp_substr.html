Assuming you are pasting this into an Argos query screen the answer is simple.  2 things:

 

1. don't include the semicolon at the end of the statement:.  This will resolve that error

2. BUT you will then likely get an error saying that it is an invalid field name.  So add an alias to your selected statement

 

select regexp_substr('joe,allen,bishop','[^,]+', 1, level)  AS RESULTS
from dual
connect by regexp_substr('joe,allen,bishop', '[^,]+', 1, level) is not null
