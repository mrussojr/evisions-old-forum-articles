Thanks Ace!...After researching this topic we tried adding:

 

 /*+ opt_param('_optimizer_use_feedback' 'false') */

 

directly after the select statement and it solved the problem. This essentially "turns off" the Oracle SQL optimizers that were slowing down the query execution on the second run attempt. The query would run okay on the 1st attempt but then time out on any attempt after that because of the SQL optimizer.
