My human resources department runs a report once a month.  Some months the report can have records and some months this report will not.  They would like a report generated even if no records are returned with a message like "none found with the date". Is this possible to do?

Here is one option:  Edit the "Execute the report" task in the schedule.  Then, check the "Fail if report query returns 0 records" checkbox and click the OK button.  Finally, on the Events tab in the schedule, change the On Error option to "Email" and configure as appropriate.

 

If they want an actual report, your report query could UNION with a dummy record so that they always get a report.
