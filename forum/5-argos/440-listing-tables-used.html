On a few occasions, I have wanted a list of tables used in a datablock or one or more folders. To make it easy, I created an Automator(OSX) action where I can drag and drop an Argos Object Contents onto the action and it generates a sorted list of unique table names used in the datablock(s). The reason it works is because I use the visual editor as much as possible which produces an Object Contents line that lists the tables for each object.

 

Since some people expressed an interest in this I figured I would share it here. The command line version is below. It is in five parts and does the following:

1. Get all of the lines that begin with the word “Tables” from the Object Contents file.

2. Use sed to remove “Tables = “ from the lines

3. Isolate the table alias name

4. Strip the quotes

5. Return an ordered list of unique values

 

grep -n ^.*Tables.* $@ ObjectContents.txt | sed 's/Tables = //g' | grep -o '"[^"]\+"' | tr '"' ' '  | sort -u

 

This does one thing and does it well. However, the concept can be applied in many ways. Other ones on my list to create include one that returns a list of functions used and one that can return tables used in subqueries.
