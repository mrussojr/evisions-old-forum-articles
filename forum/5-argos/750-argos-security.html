We at Clark University has implemented the proxy connection to Banner. MAPS needs to be on 6.1.1. It is beneficial to us because we setup SSO and don't use LDAP, it requires users to have correct and valid Banner password in MAPS. Once we setup the banproxy connection, our users are able to connect to Banner database without the need to enter their passwords. MAPS doesn't care if the user's password has changed. When users connect to Banner database via the banproxy connection, the user owns the database session so that database securities are in action and audit is possible.

Don't use Banproxy.  but for security we use several strategies depending on the situation

Dashboards are grouped into folders unless they are viewable by any employee.

 

Folders have security set up for specific users in a specific area (student services, registration, business office etc.) 

 

Specific dashboards, or specific elements may be restricted further to a single or specific subgroup of users by having a SQL variable set up to check the user and only allow the element to be visible if the user is in the approved list.

 

It is also possible to have Argos check the Banner permissions for specific things such as budget and HR roles.
