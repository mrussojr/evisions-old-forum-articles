I am trying to use a with statement in a multicolumn list box but I don't get any results.  The code runs fine in SQL developer and gives me results.  What am I missing?

 

 

with spriden_test as (select spriden_id, spriden_first_name from spriden where spriden_id = 'T00033327')

select s.spriden_id as employee_name from spriden s, spriden_test st
where s.spriden_id = st.spriden_id

Wrap it up as an inline view.  Argos allows DML statements that only start with SELECT.

 

Select *

from

(

with spriden_test as (select spriden_id, spriden_first_name from spriden where spriden_id = 'T00033327')

select s.spriden_id as employee_name from spriden s, spriden_test st
where s.spriden_id = st.spriden_id

)

FYI... I found that the "with" query works fine in the datablock as long as the recordset box is checked , and the datablock is being used hands on by a user.. vs run from a schedule. It seems that when run from a schedule via the MAPs server, things are interpreted differently and we'd get an ORA error related to the connection and only DML queries being allowed.  That is where the "wrap as an inline view" suggestion came in... I added the wrap as suggested and all worked great.
