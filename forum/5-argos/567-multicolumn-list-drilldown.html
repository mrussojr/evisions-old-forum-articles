Hi Pavan.  Since the multi-column list does not have an OnClick event, then you cannot do that directly.  However, you can use a button or a label to move to another form. 

 

Forms can access data from other forms.  For example, if you have a MultiColumn1 on Form A, elements on Form B can still access MultiColumn1.  You could also have SQL variables that use MultiColumn1.  HTH

Interestingly, I came to this 'community' to look for info on Multi-column selects. My plan is to do what (I believe you may be asking to do), and it will work when you call the 1st Drop-down using it's Variable name in the select statement.  In my first MC-Drop-down, I had the following choices:  BW, M, OL, WS.  Depending on the selection I want to show in the next Drop-Down the acceptable choices. I quick example would be: 

select decode(:p_dd_PICT.Main,'BW',1,'M',2,'OL',3,'M',4,null) as "NO"
from dual

The variable name [in bold] is pulling the selected option from the first drop-down.

If I understand correctly, you have 2 drop down lists and you want the second drop down list to populate with items that fits the criteria of first drop down list?

yeah I believe you could just reference the first dropdown list in your query for second one.

 

First drop down list: BW, M, OL, WS

Second drop down list: 

SELECT
  CASE
    WHEN :DropDown1.Variable_Name = 'BW' THEN 'NO'
    WHEN :DropDown1.Variable_Name = 'M' THEN 'YES'
    WHEN :DropDown1.Variable_Name = 'OL' THEN 'MAYBE'
    WHEN :DropDown1.Variable_Name = 'WS' THEN 'WHATEVER'
  END
FROM 
	TABLE

CASE

END
