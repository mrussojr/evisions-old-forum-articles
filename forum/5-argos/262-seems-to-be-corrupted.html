Possible issues:

-------------------------------------------------------

 

it sounds like the .exe file that gets downloaded from the E-Launcher is somehow corrupt.  I would recommend this for each DESKTOP:

navigate to the user's file location

 

C:\Users\username\AppData\Roaming\Evisions\MAPClients\Argos

 

and delete the folder for which the corrupted version matches.  Then, when they log into e-launcher again and click the Argos button it will automatically download a fresh version of that particular .exe file.  

------------------------------------------------------

 

However, if you are not still using this version (4.3.2.760) anymore then you have another issue... why are people launching this version instead of your most current version?  If this is the case then this may be what's up (I've had this happen)...

 

usually the helpdesk is "helping" when they make a shortcut on a user's desktop that points to the .exe in the location I listed above.  This is fine temporarily, however when you upgrade your 4.3 version, a new .exe is downloaded -- of which the user should NOW be using.  Unfortunately, their desktop shortcut is still pointing to the older version .exe.  You must delete any shortcuts to a static .exe and make them go through the e-launcher to access Argos.  

 

To test this, if they open their shortcut and instead of the e-launcher they get just the argos application login then you know you are dealing with an outdated shortcut that needs to be deleted.

 

Other than these two things, I cannot imagine what else it could be
