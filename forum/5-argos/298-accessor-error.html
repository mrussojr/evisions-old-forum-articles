If the full error message is "Accessor is not a parameter accessor" then take a look at Knowledge Base article # KBA24.

 

 

Description:

I get the following error when attempting to validate my query. What can I do to fix this?

 

OraOLEDB: Accessor is not a parameter accessor

 

Solution:

This error is typically reported when the Oracle provider is handling invalid dates or very large/infinite decimal places in a floating point calculation/field.

 

If the issue is with a date field, please verify that the dates being returned are valid. Sometimes values are entered into the database ambiguously and the provider doesn't know how to resolve them (i.e. 05 was entered for a year. The provider does not know how to resolve this because it could be 2005, 1905, 1805, etc.).

 

Adding the TRUNC() or TO_CHAR() functions to the fields could be helpful in both handling invalid dates or very large/infinite decimal places in a floating point calculation/field.
