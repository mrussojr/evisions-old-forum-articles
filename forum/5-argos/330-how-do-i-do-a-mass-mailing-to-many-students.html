I'm assuming you are talking about mail that uses a stamp .  

 

Some options are:

 

You'd create your report that groups on student ID or other unique student identifier, and you'd be sure to Force Page Break after in the group header band.  You would include the address in the report data, even though you might not use it on the report you want to mail.  Be sure you have a particularly unique sort order that you can replicate.  Then on a second report, you can create your labels with the address data fields, in the same order as the printed reports -- you can customize a label report in the banded report options.  Now you can have a student stuff envelopes!

 

Another idea is to create a csv data file that you use in a mail-merge document (like Word).

 

If emailing, use the burst function in a report schedule.  You don't actually have to schedule it, you can run it manually when needed.  There is good documentation about bursting both in the web help and in the user guides.

 

Good luck!
