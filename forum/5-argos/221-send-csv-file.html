This was a bug introduced in ARGOS this year.  Evisions provided us a patch to fix it when I complained that waiting for the next release was not acceptable.  The scheduled report was, in fact, remembering the selection, but when you edited again you couldn't see what had been saved.  It never did that before that version.  I'm on Version 5.3.1 Build 1878, My MAPS administrator (in a different organization) told me that this was a bug introduced in 5.3 and scheduled for a fix in 5.4, but Evisions gave him a hotfix for it.  Since you say you're on 5.1, I'm not sure why you're seeing this.  Going back to basics, do you have the Populate value by: set to By value or chosen record(s)?  If you have it set to Do not save selected value(s), it will be blank when you come back in to edit the Execute action on a scheduled report.
