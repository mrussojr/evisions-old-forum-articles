I believed this would be used:    SELECT SUBSTR(TABLE.TEXT,1,250) or some variation on a calculated field when building a block .   Does anyone have some insight?  Is my syntax incorrect, or is this not possible within the DataBlock versus the report?

SUBSTR is an Oracle function.  So, as long as you are connecting to an Oracle database and you are making the change in a SQL editor, you should be okay. 

 

For example, this would work in the datablock on the Report Query tab (or anywhere else SQL is used): 

select SPRIDEN.SPRIDEN_First_NAME AS "FirstName",
       SUBSTR(SPRIDEN.SPRIDEN_LAST_NAME, 0, 1) || '.' as "LastInitial"
  from SPRIDEN

Microsoft SQL Server has a slightly different syntax: 

select ccc_students.FirstName as "FirstName",
       SUBSTRING(ccc_students.LastName, 1, 1) + '.' as "LastName"
  from CCC.dbo.ccc_students ccc_students
