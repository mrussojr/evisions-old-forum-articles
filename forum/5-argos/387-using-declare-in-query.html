the only way I've been able to successfully declare variables is to create a custom sql variable in argos (the variables tab) that utilizes a full PL/SQL block.  Then, when a button or some action happens on the form the custom variable is called.  But then after doing this I found that my users needed access to the tables I was using, which was not viable in some cases... especially in the cases where the procedure manipulates tables.  I found it better to create the procedure on the database end and then call the procedure in the argos variable.  that way I could grant execute access for this procedure to the user without giving them direct select/update/insert privileges to the individual tables.

 

But you may be asking a much simpler question, so may I ask why you would want to use a declare statement?  Typically your query variables are set by the controls/parameters you create on your form.

I found an example of one that I did (sorry it's long but if you are like me then I like to see the whole picture!).  This is my procedure for our award letter distribution process:

 

BEGIN
IF :btnProcMan is not null THEN

 

DECLARE

FLOC varchar(150):= '\\rollins.edu\share\FinAid\Workstudy\AwardLetters\';

 

CURSOR GET_RECORDS IS

SELECT DISTINCT ZZ.PIDM,ZZ.LTR_EXIST_IND,ZZ.RNUM,ZZ.LASTNAME,ZZ.FIRSTNAME,ZZ.MIDDLENAME,
               'AWD' "TYPE",
               --'C:\AutoImport\Finaid\Data\'||ZZ.RNUM||'_AwardLetter_'||:txtAidYr||'.pdf' "FILE",
               'F:\'||ZZ.RNUM||'_AwardLetter_'||:txtAidYr||'.pdf' "OBFILE",
               ZZ.RNUM||'_AwardLetter_'||:txtAidYr "DOCNAME",
               'FAAwardLetter' "DOCTYPENAME",
               16 "FILETYPENUM",
               TO_CHAR(SYSDATE,'MM')||'-'||TO_CHAR(SYSDATE,'DD') "CSTM"
FROM
(
        SELECT
        AA.RORSTAT_PIDM "PIDM",
        CASE WHEN LTR.GURMAIL_PIDM IS NULL THEN 'N' ELSE 'Y' END "LTR_EXIST_IND",
        FF.SPRIDEN_ID "RNUM",FF.SPRIDEN_LAST_NAME "LASTNAME",FF.SPRIDEN_FIRST_NAME "FIRSTNAME",FF.SPRIDEN_MI "MIDDLENAME"


        FROM
        FAISMGR.RORSTAT AA,
        (SELECT GURMAIL_PIDM, GURMAIL_DATE_INIT FROM GENERAL.GURMAIL
         WHERE GURMAIL_LETR_CODE='FA_AWARD_LTR'
         AND GURMAIL_DATE_PRINTED IS NULL
         AND GURMAIL_AIDY_CODE=:txtAidYr)LTR,
        (select distinct CC.saradap_pidm "PIDM", CC.saradap_coll_code_1 "COLLEGE", DD.MAXDECDATE "DEC_DATE"
         from   saturn.saradap CC,
                (select AA.sarappd_pidm, AA.sarappd_term_code_entry, AA.sarappd_appl_no, AA.sarappd_apdc_code, BB.MAXDECDATE
                 from  saturn.sarappd AA,
                       (select sarappd_pidm,sarappd_term_code_entry, sarappd_appl_no, max(sarappd_apdc_date)"MAXDECDATE"
                        from   saturn.sarappd GROUP BY sarappd_pidm,sarappd_term_code_entry, sarappd_appl_no)BB
                 where AA.sarappd_pidm=BB.sarappd_pidm
                 and   AA.sarappd_term_code_ENTRY=BB.sarappd_term_code_ENTRY
                 and   AA.sarappd_appl_no=BB.sarappd_appl_no
                 and   AA.sarappd_apdc_date=BB.MAXDECDATE)DD
         where  CC.saradap_pidm=DD.sarappd_pidm
         and    CC.saradap_term_code_entry=DD.sarappd_term_code_entry
         and    CC.saradap_appl_no=DD.sarappd_appl_no
         AND    TRUNC(SARADAP_APPL_DATE)>=TRUNC(SYSDATE-365)
         AND    SARAPPD_APDC_CODE IN ('AA','AT','IA'))ZZ,
        (SELECT SORCONT_PIDM "PIDM", MAX(SORCONT_CONTACT_DATE)"LSA_DATE" FROM SATURN.SORCONT WHERE SORCONT_CTYP_CODE='LSA' GROUP BY SORCONT_PIDM)LSA,
        (SELECT SS.SGBSTDN_PIDM "PIDM", SS.SGBSTDN_COLL_CODE_1 "COLLEGE"
         FROM SATURN.SGBSTDN SS, (SELECT SGBSTDN_PIDM, MAX(SGBSTDN_TERM_CODE_EFF)"MAX_TERM" FROM SATURN.SGBSTDN GROUP BY SGBSTDN_PIDM)MM
         WHERE SS.SGBSTDN_PIDM=MM.SGBSTDN_PIDM
         AND   SS.SGBSTDN_TERM_CODE_EFF=MM.MAX_TERM)ST,
        (SELECT RPRATRM_PIDM, SUM(RPRATRM_OFFER_AMT)"OFFER" FROM FAISMGR.RPRATRM WHERE RPRATRM_AIDY_CODE=:txtAidYr GROUP BY RPRATRM_PIDM)TOT_OFFER,
        SATURN.SPRIDEN FF

        WHERE
        AA.RORSTAT_AIDY_CODE=:txtAidYr
        AND AA.RORSTAT_PIDM=FF.SPRIDEN_PIDM
        AND FF.SPRIDEN_CHANGE_IND IS NULL
        AND ((TRUNC(AA.RORSTAT_PCKG_COMP_DATE)>=:dtePckgComplDate)
             OR
             (LTR.GURMAIL_PIDM IS NOT NULL)
            )
        AND AA.RORSTAT_PIDM=TOT_OFFER.RPRATRM_PIDM
        AND TOT_OFFER.OFFER>0
        AND AA.RORSTAT_PIDM=LTR.GURMAIL_PIDM(+)
        AND AA.RORSTAT_PIDM=ZZ.PIDM(+)
        AND AA.RORSTAT_PIDM=ST.PIDM(+)
        AND AA.RORSTAT_PIDM=LSA.PIDM(+)
        AND (ZZ.PIDM IS NOT NULL OR ST.PIDM IS NOT NULL)
        AND (AA.RORSTAT_PIDM NOT IN (SELECT GURMAIL_PIDM FROM GENERAL.GURMAIL WHERE GURMAIL_AIDY_CODE=:txtAidYr AND GURMAIL_DATE_PRINTED>=:dtePckgComplDate AND GURMAIL_LETR_CODE='FA_AWARD_LTR')
             OR LTR.GURMAIL_PIDM IS NOT NULL)
        AND ((((ZZ.COLLEGE=:lstCOLLEGE.CODE AND ST.COLLEGE IS NULL)
                OR
               (ST.COLLEGE=:lstCOLLEGE.CODE AND ZZ.COLLEGE IS NULL)
                OR
               (ST.COLLEGE=:lstCOLLEGE.CODE AND ZZ.COLLEGE IS NOT NULL)) AND (AA.RORSTAT_PGRP_CODE<>'ED' OR AA.RORSTAT_PGRP_CODE IS NULL))
             OR AA.RORSTAT_PGRP_CODE=:lstCOLLEGE.CODE)
        AND ((ZZ.COLLEGE IN ('AS','CR') AND ST.COLLEGE IS NULL AND LSA.LSA_DATE<SYSDATE)
              OR
             (ZZ.COLLEGE IN ('HH') AND ST.COLLEGE IS NULL AND ZZ.DEC_DATE<SYSDATE)
              OR
             (ST.COLLEGE in ('AS','HH','CR'))
              OR LTR.GURMAIL_PIDM IS NOT NULL)
        AND :chkProcessAll='Y'


        UNION
        SELECT AA.RORSTAT_PIDM,
        CASE WHEN BB.GURMAIL_PIDM IS NULL THEN 'N' ELSE 'Y' END,
        FF.SPRIDEN_ID,FF.SPRIDEN_LAST_NAME "LASTNAME",FF.SPRIDEN_FIRST_NAME "FIRSTNAME",FF.SPRIDEN_MI "MIDDLENAME"
        FROM FAISMGR.RORSTAT AA,
             (SELECT DISTINCT GURMAIL_PIDM,GURMAIL_AIDY_CODE FROM GENERAL.GURMAIL
              WHERE GURMAIL_LETR_CODE='FA_AWARD_LTR' AND GURMAIL_DATE_PRINTED IS NULL AND GURMAIL_AIDY_CODE=:txtAidYr)BB,
             SATURN.SPRIDEN FF,
             (SELECT RPRATRM_PIDM, SUM(RPRATRM_OFFER_AMT)"OFFER" FROM FAISMGR.RPRATRM WHERE RPRATRM_AIDY_CODE=:txtAidYr GROUP BY RPRATRM_PIDM)TOT_OFFER
        WHERE AA.RORSTAT_PIDM=:mtcLetterList.PIDM
        AND AA.RORSTAT_PIDM=FF.SPRIDEN_PIDM
        AND AA.RORSTAT_PIDM=TOT_OFFER.RPRATRM_PIDM
        AND TOT_OFFER.OFFER>0
        AND FF.SPRIDEN_CHANGE_IND IS NULL
        AND AA.RORSTAT_PIDM=BB.GURMAIL_PIDM(+)
        AND :chkProcessAll='N'
)ZZ
;

TEMP_REC GET_RECORDS%ROWTYPE;

 

BEGIN

DELETE FROM ARGOS.GTT_PIDMS;

OPEN GET_RECORDS;
LOOP
    FETCH GET_RECORDS INTO TEMP_REC;
    EXIT WHEN GET_RECORDS%NOTFOUND;

    INSERT INTO ARGOS.GTT_PIDMS VALUES
    (TEMP_REC.PIDM);

    IF :chkPrintDate = 'Y' THEN
       ARGOS.UPDATE_GURMAIL_FA_AWARD_LETTER(TEMP_REC.PIDM,:txtAidYr,TEMP_REC.LTR_EXIST_IND);
       --INSERT INTO ARGOS.NOLIJ_AUTO_IMPORT VALUES(SYSDATE,TEMP_REC.PIDM,TEMP_REC.TYPE,TEMP_REC.FILE,TEMP_REC.CSTM,:txtAidYr,FLOC);
       INSERT INTO ARGOS.ONBASE_AUTO_IMPORT VALUES(TEMP_REC.PIDM,TEMP_REC.RNUM,TEMP_REC.LASTNAME,TEMP_REC.FIRSTNAME,TEMP_REC.MIDDLENAME,:txtAidYr,TEMP_REC.DOCNAME,TEMP_REC.DOCTYPENAME,TEMP_REC.FILETYPENUM,TEMP_REC.OBFILE,SYSDATE);
    END IF;

END LOOP;


CLOSE GET_RECORDS;

 

END;

END IF;

END;

hmmm, how about:

 

DECLARE

outputValue int;

 

BEGIN

 

EXEC  P_LU(:$User.Name, 'A','T', :outputValue);

 

SELECT outputValue FROM DUAL;

 

END;

 

I haven't tried this, so not sure if it will work.  You still have to get the value out of your argos variable so I'm think the SELECT will work for that.  OTHERWISE... your other option is a longer process if the above does not work...

===================================================================================================================================

 

Getting the value of an OUT parameter in an oracle stored procedure is tricky, as you have to use the SET SERVER OUTPUT ON with the dbms_output.putline () statements and I don't know how argos handles those.

 

Another option would be to utilize a global temp table.... you would store the 1 or 0 value in your temp table, and then use the temp table in your select queries or wherever you need it.

 

run this on the database to create the table only once (you can re-use global temp tables):

 


CREATE GLOBAL TEMPORARY TABLE MyTempTable (


   outputValue int


) ON COMMIT PRESERVE ROWS;

 

  

 

then your procedure would look like this (notice I am not using an OUT parameter in this procedure):

 

create or replace procedure PL_U(param1 varchar, param2 varchar, param3 varchar)

  begin

      delete from MyTempTable;
 

      do some stuff with param1, param2, param3 to get your 1 or 0 value;
     
      insert into MyTempTable (outputValue) VALUES (1);   -->this would be a 1 or 0 depending upon the result of your processing
      commit;

  end;

 

then your argos variable might look like one of these two scenarios:

 

 

BEGIN

 

  P_LU(:$User.Name, 'A','T');

 

END;

 

OR

 

DECLARE 

myOutputValue int;

 

BEGIN

 

P_LU(:$User.Name, 'A','T');

select outputValue into myOutputValue from MyTempTable;

 

END;

 

 

if you use the first procedure example, you can just use the temp table in your report query.  like so:

 

select A, B, C

from ABCTable

where A in (select outputValue from MyTempTable)

 

or create a new variable or control:

 

select outputValue

from MyTempTable
