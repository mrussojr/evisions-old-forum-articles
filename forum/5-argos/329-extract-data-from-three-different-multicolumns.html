I've had issues with trying to use charts that haven't been "activated" (clicked on) in combination with other variables that DO have values.  It seems that if the chart or one of the bars is not activated by clicking, it nulls out the whole query.  Once the chart is activated via clicking, I have no issues moving forward.  So, if the user is required to click on a chart before they see data in the three multi-columns, you should have no problem.  

 

Also, NULL values can be an issue in different capacities when using charts or parameters.  It's like the initial value is not really "null", so the IS NOT NULL evaluation doesn't work.  So, in these cases, you want to avoid using null values and set variables with some value (even if it is 0) that you can use in your case stmt (like, case when :parm = 0 then... else...).  Maybe instead of NULL you could replace the value with an empty string. like '', to at least give your parameter an initial value.  Not sure how you'd do that for a chart though, but you could do this for your drop down boxes and multi-column boxes.  For my drop-down boxes, I always have an initial value that auto-selects where it says something like 'Select a Group' -- it has a Code and a Desc  column -- the code is 0 or * and the description says 'Select a Group'.  I then union that to my actual drop down values such that it  always appears at the top by sorting on the code and my drop down never has a 'null' value.

Basically, when the Run button is clicked, I (re)set the three different stuid variables to 0. They look like this:

 

select
      CASE
          WHEN :Display_StudentBreakout_CBM.stuid = '' OR :Display_StudentBreakout_CBM.stuid IS NULL
                     OR :Display_StudentBreakout_CBM.stuid = 0 OR :parm_DD_TestArea.TestName NOT LIKE 'Easy%' THEN 0
                ELSE :Display_StudentBreakout_CBM.stuid
END AS TestCBMStuid

 

(each variable of course referencing a different test)

 

I created a fourth variable that extracts the "right" stuid:

 

select
CASE WHEN :f1_SQL_ExtractCBMStuid.TestCBMStuid = 0 AND :f1_SQL_ExtractDIBELSStuid.TestDIBELSStuid = 0 THEN :Display_StudentBreakout_STAR.stuid
     WHEN :f1_SQL_ExtractCBMStuid.TestCBMStuid = 0 AND :f1_SQL_ExtractSTARStuid.TestSTARStuid = 0 THEN :Display_StudentBreakout_DIBELS_Reading.stuid
     WHEN :f1_SQL_ExtractDIBELSStuid.TestDIBELSStuid = 0 AND :f1_SQL_ExtractSTARStuid.TestSTARStuid = 0 THEN :Display_StudentBreakout_CBM.stuid
ELSE '0'
END
AS TestStuid

 

This variable is Reset upon clicking Student Detail, and the correct stuid is passed to the subsequent demographics page, and the correct data is pulled for the correct student.  This works very well in Test, but seems to be moot, since it doesn't actually work in Web View.   Grr.
