I'm hoping there is a simple solution that I'm just overlooking.  What I need is a report in Excel format that includes formatting such as bold and italic font.  Here's sample of what the output should look like:

Heading/Group 1

               Heading/Group 2

                             Heading/Group 3

                                             Detail 1 (conditionally print when not null)

                                             Detail 2 (conditionally print when not null)

                                             Detail 3 (conditionally print when not null)

 

Note the bold heading rows and italic details. 

 

What is the best way to go about this?  If we want the output to include the bold and italic font, that rules-out CSV and Extract reports, right?  If not for that, I would think a fixed-width extract report would be the best fit.  Since we need the formatting, a Banded report seems to be the way to go, but I'm having trouble getting things to display correctly when saved as Excel.  I got the data to display correctly (in PDF format) using 3 Group bands, a Detail band, and 2 Sub-Detail bands, but when I save to Excel, it doesn't keep the data in consistent columns. How does the banded report decide which Excel column to put the data in? Another sticking point is that there doesn't seem to be a way to run a Banded report in Excel format from the Web Viewer.  Is that true?

Blaine and I exchanged some more information on this problem.  The best solution we could come up with addressed the issue of inconsistent rows/columns but did not address the issue of font formatting.  As Blaine said, the options to save Banded Reports as XLS or HTM or RTF don't translate row/column positions accurately when imported into Excel.  The good news is that applying the desired formatting in Excel requires only four clicks of the mouse. 

 

An example solution is referenced below.  The source data required an UNPIVOT operation so the datablock requires an Oracle 11g or higher connection.  It does not query any Banner data.

 

 

An untested option could be to embed HTML tags in the applicable fields.  When the text file is generated then imported into Excel, a macro/script could be executed to read the tags and apply text formatting.  The tradeoff would be the macro/script becomes another thing you have to manage/deploy.  Here's a link to a discussion about "HTML Text with tags to formatted text in an Excel cell"
