Hello All,

One of our users is having an issue with how List Boxes are behaving with Argos dashboards.

When prompted with a list box that can allow for multiple selections, the user has to click items a few times in order for the selection(s) to start occurring.

As well, the refresh rate (drawing of the items) of the list box appears to become out of line/sync with the other items in the list. Eg. items start to overlap each other.

And last, when the user moves to make additional selections of other items on the form the selections of the list box appear to become deselected.

This does not occur when using the webviewer.

I've tried other browsers to launch Argos but still have the same issue.

Note, this appears to happen to only one user.

Has anyone else seen this? Is this a known bug that requires possibly a system library update?

Thanks,

Gerry

I submitted a ticket back in July 2016 for something that sounds similar to what you described.  My ticket was 113911 and titled "After upgrade to Version 5.0 Build 1283, in full Argos client, parameter selections do not always highlight in blue if PC is not using a Windows 7 Aero theme" .  At that time Evisions created a bug report named AR-4297.

 

It appears it was eventually resolved with release 5.2...  "User interface - When Windows is configured to use the 'Windows Classic' theme, the Argos client will no longer use themed styling. This resolves several display issues that were occurring when theming was not enabled in Windows. - AR-4248"

 

Hope that helps,
