I have a dashboard that allows the user to choose items from a list box and also an item from a drop down box.  I want to make the listbox criteria optional, so I changed my Where Clause to look like this:

 

     where ((a.agbgift_amt_tot = :ListBox1.Main) or (:ListBox1.Main is null)) and ((c.apbcons_pref_clas = :DropDown1.Main) or (:DropDown1.Main is null))

 

instead of this (this one works for multiple selections, but it doesn't allow no selection)

 

     where (a.agbgift_amt_tot = :ListBox1.Main) and ((c.apbcons_pref_clas = :DropDown1.Main) or (:DropDown1.Main is null))

 

When I make the change, and try to select multiple values in the listbox, I get the following error:

 

"Multiple values exist for variable "ListBox1.Main", however the SQL script has been written to expect only one value for this variable.  To accept multiple values for variables only "=" and "<>" can be used for comparison operators.  The query used the operator ">"."

 

Can someone explain why I'm getting this error or offer an alternative solution to being able to accept optional, multiple criteria?

 

Thanks in advance.

I think the error is because you're using ":ListBox1.Main is null" which isn't an =, <, or >. It has to do with how Argos interprets that behind the scenes because a listbox could have multiple values selected. Usually for an optional listbox, I add a default selection up top (like NONE or ALL) and set the box to AutoSelect it. Then in the WHERE, I have it check if :ListBox1.Main = 'NONE' (or whatever my default value is).  Hope that helps.  --Lynne
