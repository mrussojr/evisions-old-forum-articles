I couldn't figure out how to do it with just the listbox variables (like... SELECT :main_LB_Test.STVTERM_CODE FROM DUAL as that failed as soon as I tried to select 2 things in the listbox). So, I tried combining it with the original table in the listbox to get it to work. Kind of a round about way to do it and not particularly elegant, but it might get you started on a better solution.

 

For my test, the listbox (main_LB_Test) contained term codes using:
 SELECT STVTERM_CODE FROM STVTERM

 

Then I created a SQL Variable (main_SQL_Test) to concatenate all the selections from main_LB_Test: 
 SELECT rtrim( xmlagg(xmlelement(e, STVTERM_CODE || ', ')).extract('//text()'), ', ') AS THELIST
 FROM STVTERM
 WHERE STVTERM_CODE = :main_LB_Test.STVTERM_CODE

 

I placed a Data Aware Label and pointed it to :main_SQL_Test. As I selected multiple items in the listbox, the list would appear in the data-aware-label. Maybe you could do something similar and instead of a data-aware-label you could place the SQL variable in your Extract report? (I don't know enough about Extract reports to be sure, but it seems like it should work.)  

 

-- Lynne
