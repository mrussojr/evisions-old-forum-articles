Good Afternoon

 

I found a work around if anyone else was having a similar issue

 

I created a calculated field in the data block instead of using the field alone  see below

 

Table.Field_name||chr(9)

 

Apparently concatenating the value chr(9) after the field name allowed it to come through the .csv file like i wanted

We had issues with the "Argos to Excel" translation mechanism for CSV files as well. In our case, we had some columns that stored number as text and no matter what, Excel will chop off the leading zeros. This was our solution:

 
'="'||column_name||'"'
That is: Single quote, equal sign, double quote, single quote, double pipe, column name, double pipe, single quote, double quote, single quote.
