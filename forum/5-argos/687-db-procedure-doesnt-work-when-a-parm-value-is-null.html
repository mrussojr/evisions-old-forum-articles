Hi Dave,

 

What does your DB proc declaration look like? I think your input variables on the proc need to read:

 

stored_proc(var_name IN DATATYPE default null, [etc])

 

and when you call it in your SQL variable from Argos you still need to specify which parameter is sending the value, even if it null:

 

begin

 

stored_proc(:argos_var);

 

end;

It could depend on your situation, but I ran into a similar issue. I have a memo object on a dashboard that allows user to add a comment. I had to change the 'Required' property for the object to 'No' and it would allow NULLs to be passed through to the DB. Hopefully that helps!
