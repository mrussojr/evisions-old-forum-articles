We have a similar structure in place as Ace currently. We have a Development Folder and a Production folder and developers can freely edit items in their respective area. When a report in Production needs to be edited, our Argos Admin will copy it to an area where the developers can edit it. Developers are required to use the "notes" field to document changes. When it needs to be "deployed", our Argos Admin will take the staged-to-production copy, place it in the Production folder and then delete the old one. There are better ways to do this now inside Argos (namely, version control) but at the time when we implemented, this was not an option.
We have two Admins and about a dozen report writers/ datablock designers.
