I've created a pie chart using this SQL query (I'm a newbie to Argos, so I hope this isn't a dumb question!):

select COUNT(*) CountStudents, Status from
(
select
CASE
 WHEN :parm_DD_TestArea.TestName like '%Fall' then Fall_composite_status
 WHEN :parm_DD_TestArea.TestName like '%Winter' then Winter_composite_status
 ELSE Spring_composite_status
END as Status
from vw_dibels_composite_reading
where vw_dibels_composite_reading.dst = :parm_DD_District.District
and vw_dibels_composite_reading.sch = :parm_DD_School.School
and vw_dibels_composite_reading.year_of_test = :parm_DD_SchoolYear.syr) a
where Status <> ''
group by status

 

The statuses in this case are "At or above benchmark" "Below Benchmark" and "Well Below Benchmark." I would like to show "At or above" in green, "Below" in yellow, and "Well Below" in red in the chart. Is this possible?

I don't think you can set the colors manually. The closest I've come is to choose a particular color palette which has the colors in the order you need. However, note that the colors in the Argos Web Viewer are different from the Argos client. So, if you have some users with the Web Viewer and some with the Argos Client, it's unlikely you can get both to have the same color scheme. 

 

You may have to manipulate the order of your statuses to fit a pre-defined palette. Once I even added a fake entry to force things into the colors I wanted. For example, the Solid color palette goes Blue - Red - Green - Gold in the Argos Client (and blue- light blue - orange - light orange in the web viewer, sigh). If your statuses are in the order you listed, then At will be blue, Below will be Red, Well-Below will be Green.To get them to be the correct color, you'd need to first add a fake status with a value 0 -- this will consume the Blue pie segment which you don't want but it'll be a 0 so it won't show. Then put your Well-Below (Red), At-Above (Green), and Below (Gold). 

 

If I don't want pie slices in alphanumeric order, I often add an extra field to my SQL for charts-- I just call it PRINTORDER and I use CASE or DECODE or whatever to populate it with a number indicating which pie slice I want (for example, make Well-Below number 2 to force it into the Red slice), then ORDER BY PRINTORDER. 

 

For example, this is one where I'm forcing Male to be the Blue pie slice:

 

SELECT PRINTORDER AS "PrintOrder", GENDER AS "Gender", COUNT(PIDM) AS "NumStu"
FROM (
SELECT DECODE( ( SELECT SPBPERS.spbpers_sex
                 FROM SPBPERS
                 WHERE SPBPERS.spbpers_pidm = SGBSTDN.sgbstdn_pidm ),
                 'M', '1',
                 'F', '2',
                 '3'
              ) AS "PRINTORDER",

       DECODE( ( SELECT SPBPERS.spbpers_sex
                 FROM SPBPERS
                 WHERE SPBPERS.spbpers_pidm = SGBSTDN.sgbstdn_pidm ),
                 'M', 'Male',
                 'F', 'Female',
                 'Not specified'
              ) AS "GENDER",
       SGBSTDN.sgbstdn_pidm AS PIDM
FROM SGBSTDN

WHERE -- Want student data pertaining to what was in effect for the specified term.
      SGBSTDN.sgbstdn_term_code_eff = ( SELECT MAX(getmax.sgbstdn_term_code_eff)
                                        FROM SGBSTDN getmax
                                        WHERE getmax.sgbstdn_pidm = SGBSTDN.sgbstdn_pidm
                                          AND getmax.sgbstdn_term_code_eff <= :main_DD_Term.TermCode)

      -- Want only enrolled students
  AND f_enrolled_this_term(SGBSTDN.sgbstdn_pidm,:main_DD_Term.TermCode) = 'Y'
  AND f_registered_this_term(SGBSTDN.sgbstdn_pidm, :main_DD_Term.TermCode) = 'Y'
  AND :gender_BT_GetData IS NOT NULL
ORDER BY 1
)
GROUP BY PRINTORDER, GENDER
ORDER BY PRINTORDER

 

It's not pretty, I know. And, like I said, you need all your users using the same viewer (web vs client) to have a hope of getting it to work. There used to be an Advanced option for editing chart details, but mine's grayed out now. (or maybe you have to be using a certain chart type like Line Charts for it to work?) -- Lynne
