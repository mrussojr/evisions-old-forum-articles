I had hoped to be able to do additional filtering on a report using a value selected by the user at the time the report is run, that is not used by the WHERE clause in the datablock. I set up the datablock to list graduation information for a variety of reports, and only this report needs to filter by which ceremony (if any) the student will attend as well as by campus. I can't find a way to do the additional filtering based on the variable, however, only on fields in the database.

 

Since I couldn't figure that out, I set up an expression for Only print if... on each band, which works fine for the Detailband, but not so well with the headers. Header 1 (the major) only prints if a certain field is not blank, but Header 2 (the degree) should always print as long as there are students with that degree.

 

If I only select one campus and the ceremony for that campus, everything seems OK. But in the rare case that a student from one campus chooses to walk in the ceremony at the other campus, I wanted the user to be able to select both campuses but only one ceremony. What ends up happening, though, is if both campuses are selected, it is possible for no header to print for a student, so they appear to be in a previous group. I'm guessing that the first student in the new group is from the other campus, so Header2 thinks it shouldn't print, and by the time the report gets to the student who does print, it's past the logic about printing headers so just prints the detail line (this happens to be one where Header 1 should not print, though I don't know if that makes a difference). It's possible that in production, by the time of the ceremony, there will be enough in every category that this won't be an issue, but I don't want to assume that. There are a couple of degrees that we award rarely.

 

If I could do the filtering for the report as a whole, I think it would work a lot better. I really would prefer not to create a whole new datablock just for this report, because it's a really complex setup, and I don't want to have to remember to update two different datablocks if anything should change in the setup.
