As far as the checking valid data, I'd set the where clause of the update Query to verify the value e.g. "where EditBox1.Text in ('Y','N')" so it will only update if the Text is one of your options.

 

In terms of the warning box:  I saw a presentation at Cohesion Baltimore this year about displaying a warning box in argos - You could create a Panel with a text box that says "Your data didn't pass  validation" and an "OK" Button. 

 

You can set up a variable called "WarningVisible" of

 

SELECT CASE WHEN UPPER(:EditBox1) IN ('Y','N') THEN 0 ELSE 1 END AS "ShowError" FROM DUAL  WHERE *:MainButton* IS NOT NULL

 

and set the visibility of the warning box equal to WarningVisible.ShowError.    For the OK Button on the error box, set an onclick event to refresh the WarningVisible Variable (which will set it to null, and cause the warning box to disappear).

 

I hope that helps!
