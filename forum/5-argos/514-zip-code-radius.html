Posted February 21, 2018

If you are Banner institution, I am not aware of any baseline functionality to do this.  We are a Banner institution but we use a third-party product from Runner Technologies called CLEAN_Address which allows us to perform the kind of search you describe.  Built in to CLEAN_Address is a basic geo-coding accuracy level of up to the 5-digit ZIP code.  Geo-coding identifies the latitude / longitude coordinates for an address.  This helps compute distance and bearing, and to perform radial searches.

 

Additionally, you may subscribe to the “GeoCoder” option to increase geo-coding accuracy to the 9-digit ZIP-plus-four level.  This gives very accurate lat / long coordinates as well as census tract / census block identifiers, which are accurate to about the center of a city block. 

 

The “MatchMaker” GeoCoder option provides geo-coding accuracy interpolated to near rooftop levels.

I actually created an Argos Dashboard that takes in a user inputted zip code, and finds all the zip codes within a user-specified radius.

 

We use Zip-Codes.com to get zip code data such as latitude and longitude. I created a Pentaho ETL job that automatically updates a table in our database on a monthly basis with any changes to national zip code information.

 

If you have zip code latitudes and longitudes, you can calculate zipcodes within a certain radius using some math. Here is the query, where ":zip" is the user-entered zip code, ":radius" is the user entered radius in miles, and ":Search" is the search button:

  

select * from (
with coordinates as (
SELECT DISTINCT zipcode
, to_binary_float(latitude) lat1
, to_binary_float(longitude) long1
FROM va_zipcodes
where zipcode = :zip
)
, distance as (
SELECT z.zipcode,
(3958.75 * ( atan(sqrt(1 - POWER(((sin( C.lat1/57.2958) * sin(to_binary_float(latitude)/57.2958)) +
(cos( C.lat1/57.2958) * cos(to_binary_float(latitude)/57.2958) * cos((to_binary_float(longitude)/57.2958) - ( C.long1/57.2958)))), 2)) /
((sin( C.lat1/57.2958) * sin(to_binary_float(latitude)/57.2958)) + (cos( C.lat1/57.2958) * cos(to_binary_float(latitude)/57.2958) *
Cos((to_binary_float(longitude)/57.2958) - ( c.Long1/57.2958))))))) as DistanceInMiles
from va_zipcodes z
, coordinates c
)
SELECT DISTINCT z.zipcode
, trunc( abs (d.distanceInMiles), 2) as Distance_In_Miles
, z.city
, z.state
, z.county
, z.areacode
from va_zipcodes z
, distance d
WHERE D.zipcode = z.zipcode
AND ABS (D.distanceinmiles) <= :radius
and :Search is not null
)
order by Distance_In_Miles
