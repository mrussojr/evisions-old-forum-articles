Ed,

 

We have information similar to what you show, but on separate reports (so the org summary and its breakdown are one report, open encumbrances are on another). I've never created a report with unequal size columns, although I have created a report for multiple, mostly-unrelated bands before using either datasets or unions and filters on the bands. I've also structured my data so that I the report could be more free-form in terms of design e.g. instead of summing data with bands, I've done the summation inside the query then just put these fields were needed .The downside of this approach is it really needs a fixed number of data elements to work well.

 

My best suggestion is to structure your data so that the detail band is of the transactions, and then add group headers to get the roll-up amounts. You can then do a subdetail for the open encumbrances.  For example, your detail would be of the 6001- Full-time admin rows, and then there would be a group header above that that totals. I attached some pictures of each approach.

 

Hope this helps.
