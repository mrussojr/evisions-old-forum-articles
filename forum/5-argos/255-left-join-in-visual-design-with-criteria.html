In the visual designer you will want to do a combination of the (+) and the left join on the table.

 

Join table with a left join in the visual part. Then in your WHERE you will make the table calculated and for the field enter it with a (+) after the field you are joining. Then in the condition you will enter the >= and the field you are joining on.   

 

the WHERE will end up looking like this: 

and/or 

<calculated>

SCBCRSE.SCBCRSE_EFF_TERM (+)

>= SSBSECT.SSBSECT_TERM_CODE

 

 

Hope this helps.

Did you mean to quote as a bump to the thread? When looking at your original code, it's probably not going to let you use the (+) indicators because you are already using ANSI joins (inner join). If you change the table's join to be "left outer join" then you can specify any criteria you want in the "on" section.

 

For example: 

select *
from scbcrse
left outer join ssbsect on ssbsect_subj_code = scbcrse_subj_code
                        and ssbsect_crse_numb = scbcrse_crse_numb
                        
where scbcrse_eff_term = (select max(s.scbcrse_eff_term) from scbcrse s
                            where s.scbcrse_subj_code = scbcrse.scbcrse_subj_code
                              and s.scbcrse_crse_numb = scbcrse.scbcrse_crse_numb);

The second that you start limiting the query of an outer joined table in the where clause, it will force it to be an inner join. For example:

 

If I have two tables that look like this: 

Table - data
ID		DESCR
1		Test1
1		Test2

Table - moreData
ID
1
2

If I query them like this: 

select *
from moreData
inner join data
on data.ID = moreData.ID;

then I only get 2 rows because there is no ID of 2 in the data table.

 

If I change it to an left outer join, the output looks like this: 

moreData.ID		data.ID			DESCR
1			1			Test1
1			1			Test2
2			(null)			(null)

 Now even though I am outer joining the tables and getting all the rows from the moreData table, if I add a condition to the where clause that limits the data table, it goes back to being an inner join: 

select *
from moreData
left outer join data
on data.ID = moreData.ID
where data.DESCR = 'Test1';

Output of:
ID	DESCR
1	Test1

If you move that condition into the "on" section to limit it in the join itself, it will still give you all of the rows from the first table (moreData) and only join "on" the condition you set: 

select *
from moreData
left outer join data
on data.ID = moreData.ID
and data.DESCR = 'Test1';

Output of:
moreData.ID	data.ID		DESCR
1		1		Test1
2		(null)		(null)
