I don't know about doing it as the report is built.... But when I want to convert separate record items into one field with all the values concatenated together, I usually use XMLAGG to do it. So, I would add something like this to my main query's SELECT statement:

 

     -- Student Attributes as a list
        ( SELECT rtrim( xmlagg(xmlelement(e, SGRSATT.sgrsatt_atts_code || ' (' ||
                                             NVL2 ( SGRSATT.sgrsatt_atts_code,
                                                    ( SELECT STVATTS.stvatts_desc
                                                      FROM STVATTS
                                                      WHERE SGRSATT.sgrsatt_atts_code = STVATTS.stvatts_code),
                                                     '')
                                             || '), ')).extract('//text()'), ', ') AS THELIST
           FROM SGRSATT
           WHERE SGRSATT.sgrsatt_pidm = SPRIDEN.spriden_pidm
             AND SGRSATT.sgrsatt_term_code_eff = ( SELECT MAX(getmax.sgrsatt_term_code_eff)
                                                   FROM SGRSATT getmax
                                                   WHERE getmax.sgrsatt_term_code_eff <= :main_DD_Term.TermCode
                                                     AND getmax.sgrsatt_pidm = SGRSATT.sgrsatt_pidm)
           GROUP BY SGRSATT.sgrsatt_pidm
         ) AS "StuAttributes",

 

 

This produces a StuAttributes field with values like:

  EXIT (NURS Competency Exit Exam), SHBT (SHIP Between Age), P2CC (P/G2 Certificate, < Assoc), P1CC (P/G1 Certificate, < Assoc), FYAS (First Yr Seminar Exemption)

 

It can be edit to get just the codes instead like:
  EXIT, SHBT, P2CC, P1CC, FYAS

Assuming you have multiple SGRSATT_ATTS_CODEs for that max effective term, the reason you are only getting 1 attribute is because the queries are separated in the extract report builder. You should instead join the table into your report query or make a dummy report query (select 1 test from dual) and move the whole thing into the extract report as 1 piece.

Merging the two will give you multiple rows for each student if they have greater than 1 ATTS_CODE where the data in your report query is duplicated on each row. If you are looking for just 1 row with a list of values, you can do something like what Lynne Hoatson said. I would personally use a LISTAGG function instead of xmlagg because I find it easier to create and understand, but that doesn't really matter.
