Hi David, from what I could tell (wasn't sure what data source or table you were using) it looks like you have a column called Value in your report query that is a long string of emails that belong to a certain schedule.  I'm also assuming that some of your email strings in that Value field are longer than 255 chars.

 

I ran a test of my own using SQL Server as my data source and created a table field with 400 characters.  I then created a report similar to yours which outputs to a .csv file.  When I saved my report it launched in Excel and all 400 characters were included.  I also output the report to a .txt file and opened it in notepad -- all 400 characters were present.

 

The issue is not that the csv report can't handle more than 255 characters from what I could see.  I assume the issue is with the table itself or the data source driver. Sometimes odbc drivers have limitations and quirks that you have to account for, or it could be that the table field itself only holds 255 characters.  Those would be the next two things I would investigate.

 

Also, I noticed that in your report query, you have cast the field's value to varchar2(50).  This will limit the field output to 50 characters... but that may be something you were in the process of testing.

 

Hope that helps,

C
