We have successfully added hyperlinks to our banded reports.  If you place them "just so" they will render in Adobe and in Chrome/IE/Safari (Firefox works depending upon your plug-ins).  

We have inserted both dynamic hyperlinks (i.e., links to another Argos report using the API configuration) and static hyperlinks.  To insert a hyperlink, you must "hover" a label on top of an expression or rtf field -- which can either create the dynamic link at run-time via variables or can hold the static link.  The hyperlink field is "hidden" (like all my quotes?  lol) by making the font very small (if the URL is long) and making the font color blend in with the background (so the text is invisible).  Float a label on top, color it blue and underline it, and wa-la you have a "hyperlink" field.  The trick is to get the placement right so that the URL is visible to whatever renders the pdf and it is close enough to the label for it to look like a real link.  I use API links in our banded budget reports that are scheduled and sent to uppper leadership -- the reports are at a high level, but the API URL's allow them to link to another budget report which emulates a drill-down process.
