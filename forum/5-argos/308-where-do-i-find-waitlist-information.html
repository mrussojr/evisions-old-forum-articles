That information is the SSBSECT table in the SSBSECT_WAIT_CAPACITY, SSBSECT_WAIT_COUNT, and SSBSECT_WAIT_AVAIL columns.  My understanding is that Banner updates this information automatically with each enrollment change.

 

Actual enrollments are in the SFRSTCR table.  You can look at the SFRSTCR_RSTS_CODE column to determine waitlist or not.  You can check the STVRSTS table to know which does indicate a waitlist status.

The table behind the SSASECT form is SATURN.SSBSECT.  The 3 fields are:

SSBSECT_WAIT_CAPACITY

SSBSECT_WAIT_COUNT

SSBSECT_WAIT_AVAIL

 

There is a process that is usually set by the institution's IT area to run on a regular basis that updates the count and available fields when it updates the SSBSECT_ENRL and SSBSECT_SEATS_AVAIL fields.

 

It can be manually calculated at any time by using SATURN.SFRSTCR to count the number of students with the RSTS code that is used by your school to be WaitListed.  Ours is "WL" - check SATURN.STVRSTS to learn the code(s) being used.

 

SELECT
TERM, CRN, SUBJECT, COURSE, WL_CAP, WL_HC, (WL_CAP - WL_HC) AS WL_AVAIL
FROM
  (
    SELECT
    SFRSTCR_TERM_CODE AS TERM,
    SFRSTCR_CRN AS CRN,
    SSBSECT_SUBJ_CODE AS SUBJECT,
    SSBSECT_CRSE_NUMB AS COURSE,
    SSBSECT_WAIT_CAPACITY AS WL_CAP,
    COUNT(SFRSTCR_PIDM) AS WL_HC
    FROM
    SATURN.SFRSTCR,
    SATURN.SSBSECT
    WHERE
    SFRSTCR_TERM_CODE = SSBSECT_TERM_CODE
    AND SFRSTCR_CRN = SSBSECT_CRN
    AND SFRSTCR_RSTS_CODE = 'WL' -------------------------CHANGE TO YOUR WAITLISTED CODE
    AND SFRSTCR_TERM_CODE = '201780' -------------------CHANGE TO YOUR TERM CODE
    GROUP BY
    SFRSTCR_TERM_CODE,
    SFRSTCR_CRN,
    SSBSECT_SUBJ_CODE,
    SSBSECT_CRSE_NUMB,
    SSBSECT_WAIT_CAPACITY
  )

ORDER BY

TERM, SUBJECT, COURSE, CRN

(From my notes on wait listing)

  • There is a synergy between SFRSTCR and SFRWLNT (wait list notification table).
  • When someone is waitlisted, they appear on SFRSTCR with an RSTS code of WL and are assigned a priority code.
  • When a seat becomes available (as per SSBSECT),  a person on the WL queue on SFRSTCR will be populated in SFRWLNT.  The priority for this person on SFRSTCR is set to zero.  The status on SFRWLNT is set to 'P' (Pending).  If email works, a notification is sent to the student.
  • When the SFRWLNT date/time is exceeded, the WL expires for the student, and the status is set to "X" (Expired, can be deleted).
