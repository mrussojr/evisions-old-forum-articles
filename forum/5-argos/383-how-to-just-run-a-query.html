Sounds like you want to use Argos to run a process that is controlled by a SQL script.  You may want to even set it up to run on a schedule.

 

This could be very dangerous if the script is modifying data.  (Note that the connection must allow write privileges to the database for that to work anyway)

You would need to have some method of preventing it from running every time the datablock is opened.  Have a button to click maybe.

 

What I'd suggest is to create a SQL Variable.  

You can set some environmental and user variables in the script to control its execution (perhaps the user = you; SYSDATE is within a specific range of dates; button is not null)

 

Hope that helps.

Oh, gotcha.  You just need to create a multi-column box on the main form where the query can live and display the results. 

Double-click the box and choose SQL statement on the pop up.

paste the SQL query. REMOVE THE ; AT THE END or you will get an error.

click Next to have it validate that the SQL works (you can test the query in TOAD or SQL Developer prior to working in Argos)

Click finish.

 

You can fine tune the display by sizing the box,

On the property panel at the left you can:

size columns,

change the font,

add row lines

have the headings word wrap

etc...
