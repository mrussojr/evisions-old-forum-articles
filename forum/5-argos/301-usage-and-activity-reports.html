The MAPS Data Warehouse creates an Audit table that gives information for whenever a user executes a dashboard/report. It also shows scheduled reports, security settings, datablock moves, etc. The MAPS administrator can set the amount of days that the Audit table saves. Getting basic dashboard and report usage is easy if you use that table. Our office uses an SSIS package to import and transform the data into a SQL server database to make it easier to work with because the MAPS data warehouse uses sqllite.

 

The dashboard executes are flagged as "Dashboard". You have to be careful though because two rows are created in the Audit table for each dashboard. One is flagged as "Dashboard" like I said and the other is flagged as a "Report" but ends in ".Dashboard".

Both the scheduled and non-scheduled reports are flagged as "Report", but the scheduled reports have an Application setting of "ArgosRepRun" instead of "Argos" or "ArgosWeb."

 

Hope that helps

I have a group of developers that like to keep track of DataBlock/Report usage.  I have my auditing setup to hold 30 day worth of data.  I also have an "admin" DataBlock that I setup so that I can keep tabs on things.  Since audit details is one of the things that I need to look at occasionally, I already had a page setup for it...with the dropdown and listbox selections already available (the DataBlock is connected to the MAPS Date Warehouse).  So, I setup a report that runs off of my existing DataBlock.  The query that I use is as follows:

 

select Audit.Id AS Id,
       SysUser.Description As ActivityDesc,
       datetime(Audit.ActivityDate) as "ActivityDate",
       Audit.Application As Application,
       Audit.Action As Action,
       Audit.Text As Details
  from SysUser inner join Audit on SysUser.Name = Audit.UserName
 where ( date(Audit.ActivityDate) >= date(:parm_DE_Start)
         or :parm_DE_Start is null )
       and ( Date(Audit.ActivityDate) <= date(:parm_DE_End)
         or :parm_DE_End is null )
       and ( Audit.Application = :parm_LB_Application.Application
         or '(All)' = :parm_LB_Application.Application )
       and ( SysUser.Description = :parm_LB_Users.Description
         or '(All)' = :parm_LB_Users.Description )
       and ( Audit.Action = :parm_LB_Action.Action
         or '(All)' = :parm_LB_Action.Action )
 order by Audit.Id desc

 

My report is setup to run every 30 days and dropdown/listbox selections are set as such.  The report goes to them as a CSV file.  It essentially contains all of the audit information tied to the username.  From this, they can slice the data any way that they wish.  If this were a general user, it would be easy enough to limit the report as needed.  There's a wealth of information in the MAPS Data Warehouse. :-)

 

Hope this helps,
