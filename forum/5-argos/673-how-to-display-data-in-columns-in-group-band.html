Hello All,

I'm writing a scholarship recipient report for the owners of the scholarship fund that needs to display the recipients of the scholarship.

In order to display the recipients for each scholarship (and keep it on one page) they need to be in columns (4 per line) in the group band. The number of recipients is variable so the band size can grow.

An owner can be in charge of multiple scholarships so each letter can have multiple scholarships listed that include Market Value, Principle amt and Awarded amount.

 

Here is an example of the format for each group band :

 

Scholarship Name

Market Value:

Principal:

Awarded Amt:

 

Recipient 1          Recipient 2          Recipient 3          Recipient 4

Recipient 5          Recipient 6          ......                             ......

 

Currently I've written a function in oracle that will assign a recipient a number (add them to a specific field) and have created multiple bands with datafields that correspond to each recipient and each band will display (or not) based on the number of recipients for the scholarship.

 

So it does work but I'm wondering if there is a better way to do this? This is a functionality that can be used in multiple areas and I'm looking for a way to streamline this process.

 

Thanks,

True.  What if instead of columns, you had four different fields, in tabular format.  With a subdataset for each column # assigned to the recipient, you could technically create the illusion of columns.

Something like:

 

MY REPORT

 

my scholarship group band with its information

-----------------------------------------------------------------------------

subdataset 1        subdataset2       subdataset 3       subdataset4

 

if the recipients of the scholarship have been divided by four and given a sequence number that determines the column they fall into, the you could set the detail band height to '0' to suppress it and use subdetail bands with the subdatasets in tabular format.  The subdataset 1 query would select recipients with their column=1 and scholarship = :argosreport.scholarship.

 

The subdetail band should then become dynamic, and if the column seq number is distributed equally among the recipients (say like using dense_rank() with a calculation) then it would appear as near equal column sizes.

Unless I've over-complicated things, which I tend to do...
