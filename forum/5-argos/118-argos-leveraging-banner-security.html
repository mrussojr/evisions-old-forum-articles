Jim,

 

We have most of our Argos reports by Banner Module (Student, Financial aid, etc.) and subgroups, if necessary (student - grades) etc. Then we have a Oracle role with select on tables the datablocks needs for its query for which those modules correspond. Then users are granted those roles so they can run the reports.

 

I guess what I'm saying is we made some convenient groups based on the existing tables relationships: for example, sfrstcr, ssbsect, scbcrse are in a group (with others) and then this is assigned to any user who need to run "Student - Catalog" style reports. For easy identification, they were tagged as Argos roles in their name so that we can then query for who is in those group, etc
