David,

 

When you say enrolled, I assume you mean registered for a term.  This is the code we use to count the number of students registered for a given term:

 

select count(distinct sfrstcr_pidm) as student_count
  from sfrstcr, stvrsts
 where sfrstcr_term_code = :term
   and sfrstcr_rsts_code = sfrstcr_rsts_code
   and stvrsts_incl_sect_enrl = 'Y'

There's also 2 functions I sometimes use to determine if a student is enrolled and/or registered (the 2 aren't exactly the same). You just need to pass in a PIDM and a Term Code. They return a Y/N. Like:

 

WHERE f_enrolled_this_term(SGBSTDN.sgbstdn_pidm,:main_DD_Term.TermCode) = 'Y'
AND f_registered_this_term(SGBSTDN.sgbstdn_pidm, :main_DD_Term.TermCode) = 'Y'

 

-- Lynne

I first learned about them as I transitioned into working with SQL from the prior SQL developer. Depending what access you have to the Banner database, you might be able to write a query to see what ones are out there for you. For me, I was able to do a quick-and-dirty Argos dashboard with a drop-down that lists the functions and then a multicolumn listbox to show the function itself. If you don't have access to these system tables, you may just need to ask a DBA. 

 

I use this code to get the list of functions:

SELECT OBJECT_NAME
FROM   ALL_OBJECTS
WHERE  OBJECT_NAME LIKE 'F_%'
  AND  OBJECT_TYPE='FUNCTION'
ORDER BY OBJECT_NAME

 

Once a function is picked, this code in a multi-column listbox gives me a peek into what the function's doing:

SELECT LINE, TEXT
FROM   ALL_SOURCE 
WHERE  UPPER(NAME) = UPPER(:main_DD_Function.OBJECT_NAME)
ORDER BY LINE

 

Hope that helps!
