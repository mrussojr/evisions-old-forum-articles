I have a user request to allow them to type specific, and multiple, values into an Edit Box and filter a result set by those values.  It would make sense if I could just take the value of the Edit box and encapsulate it in Parenthesis and plug it into an SQL statement using the IN Operator, but that doesn't work.  Maybe because giving them the ability to do that might open up the database to an SQL Injection attack? 

 

So my question is, has anyone had to build a workaround for the IN operator using an Edit box or some method other than List boxes to allow users to manually enter multiple, random, parameters for a filter?

 

Thanks for any insight.

Give the user a memo box for entry on the dashboard. It will give them more space. Then go to the properties of the box and select YES for Multi Entry. They can then enter a value on each line and the result will return for each entry.
