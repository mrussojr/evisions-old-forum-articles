You can't pull data from 2 separate data sources into a single MultiColumn listbox unless you have the 2 databases dblinked at the Oracle level. This needs to be done by a DBA in Oracle and is totally outside of Argos/MAPS. 

 

You can pull data from each source into separate MultiColumn listboxes though. You can get data into MultiColumn A using the default data connection. For MultiColumn B, open the box to enter the SQL statement and near the bottom left of the dialog box, there's a link that says "Connection: <default>". Click on that and choose a different connection.

In our situation, our DBA had to set up links from one database to another.  Once that was done, we were able to code our SQL to be able to "talk" to another database from the one we were coding in.  As an example,

 

Select spriden.spriden_id,

               degreeworks.student_id            <---------- this is from the database being linked to

from

           spriden spriden,

          degreeworks.tablename@linkname.edu    <-------------- use the "@linkname.edu" to call to the linked database

 

Substitute your table and database names and you should be good to go.  Also, in our cases, our DBA did not allow links from our production databases over to test or ODSP databases, but did allow links from test database and ODSP databases to production databases.  That way, he eliminated any "drain" on produciton databases.

 

I hope this helps... it can be confusing initially, but after doing it a few times, it will make sense.
