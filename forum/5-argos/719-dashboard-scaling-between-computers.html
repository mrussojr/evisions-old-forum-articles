Hi Ian,

 

Generally, we only have one thing which aligns to client.  This object will take the remaining space.  Our typical datablock has a header panel, a parameters panel, and a results multi-column box.  The header panel is aligned to top so that it stretches to fill.  Below the header panel is a parameters panel.  This panel is also aligned to top.  We take care so that the parameters are not very wide so that does not get cut off.  Finally, we have a results multi-column box which is aligned to client.  This takes up the remaining space.

 

The advantage is that there is rarely any scrolling except for in the results area.  Attached is a picture which may help.

Hi Ian,

 

In addition to what Ace has suggested, you can also use the Height and Width of a form (in the dashboard editor) to specify the minimum window size you would expect Argos to be opened on.  I.e. if the smallest resolution you would allow Argos to run on would be 1024x768, you would specify the Width of the form to be 1024 and the Height to be 768.  This will ensure that your DataBlock Designers design within the bounds you have specified.  I included a screen shot where the Argos form uses a size of 300 for both Height and Width.  You will see dotted lines starting in the top left corner and moving down and right.  This would be the critical screen that your DataBlock Designers should design to.
