Hi Everyone,

 

 

We've identified an issue in the 5.4 release of Argos where objects that have been "sent to back" behind other objects on a dashboard are being moved to the front when any changes are made to the DataBlock.

 

We are working on a fix for this issue and will be releasing it shortly.  In the meantime, if you encounter this problem you will want to go back into the DataBlock, send the object to the back, and save.  You will need to send the object to the back again each time you edit the DataBlock.

 

If you have run into this problem please keep an eye on this forum as we will make an announcement once a fix is available.  We regret any inconvenience this has caused and thank you for your patience while we address this matter.

Hi Everyone 

 

 

Evisions is pleased to announce the release of Argos version 5.4.1, which resolves the above mentioned issue. 

 

This release is an update to the Argos client only. There are no updates to the Argos mapplet, Argos Web Viewer, or MAPS.

 

As always, we highly recommend installing updates in a test environment before applying them to your production environment.  You should make sure that a current backup is available in case of any unforeseen issues.

 

For installation instructions, please refer to the Argos 5.4.1 Release Guide.

 

If you have any difficulty with the updates, just contact our HelpDesk and we will be glad to assist you.
