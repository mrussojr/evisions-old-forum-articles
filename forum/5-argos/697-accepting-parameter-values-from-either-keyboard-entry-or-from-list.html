Currently, I have a data block that has 2 multi-list boxes.  The first list asks the user to select a term from the STVTERM table.  Based on the user's selection, a list of CRNs and the Course Description is displayed in the second list box, which allows the user to use the mouse, scroll down the list, and select the CRN.  When the 2 parameters are set, the user clicks on a Run button and the data is pulled based on the parameters.

 

The data block works as expected.  However, the user would like to be able to type in the CRN instead of scrolling down to the correct CRN since list can be quite lengthy.

 

Is there a way to allow the user to not only type in the CRN manually, but also automatically scroll down the list and  highlight the CRN in the list box and allow the user to select the highlighted CRN?

 

Here is the code to populate the list box for Term:

 

SELECT       STVTERM.STVTERM_CODE "VALUE",
                         STVTERM.STVTERM_DESC "DESCRIPTION"
FROM          SATURN.STVTERM STVTERM
WHERE      STVTERM.STVTERM_CODE >'201200'

 

And here is the code to populate the CRNs:

 

SELECT              SSBSECT_CRN "VALUE",
                                SSBSECT_SUBJ_CODE||' '||SSBSECT_CRSE_NUMB "DESCRIPTION"
FROM                 SSBSECT
WHERE             SSBSECT_TERM_CODE = :parm_LB_term1.VALUE
 ORDER BY     SSBSECT_CRN

 

Thank you for your help.

Thanks for the feedback Thomas.

 

I came up with a compromise solution to the problem.  I created an Edit box to allow the user to manually enter the CRN (into a variable name parm_LB_enterCRN) and the multicolumn list box that used to populate with all the CRNs based on the Term code now displays the CRN entered by the user.  It's not what I wanted to do, but it's what I could do for now.  There has to be a way to enter the CRN manually and have the multicolumn list box show the contents as the user enters the CRN.

 

Here's the code for the multi-column list box:

 

SELECT     SSBSECT_CRN "VALUE",
                       SSBSECT_SUBJ_CODE||' '||SSBSECT_CRSE_NUMB "DESCRIPTION"
FROM       SSBSECT 
WHERE    SSBSECT_TERM_CODE = :parm_LB_term1.VALUE
AND          SSBSECT_CRN = :parm_LB_enterCRN
ORDER BY  SSBSECT_CRN   <==== Hmm....I don't think I need this line anymore
