Over the years we have changed the names of some of our orgs a few times, in order to pull up only the latest org I created a max(eff_date) sub-query in my data-block.  This does not seem to work, the drop down block is still showing multiple orgs to select and when the 'get' data comes back it includes multiple duplicated lines for the orgs which have had changes in previous years.  I am not sure what I am doing wrong?  Any help would be appreciated, Thanks!

 

Here is the SQL view:

 

select FGBOPAL.FGBOPAL_ORGN_CODE "Org",
       FTVORGN.FTVORGN_TITLE "Title",
       FTVORGN.FTVORGN_ORGN_CODE||' - '||FTVORGN.FTVORGN_TITLE "Display"
  from FIMSMGR.FGBOPAL FGBOPAL,
       FIMSMGR.FTVORGN FTVORGN
 where ( FGBOPAL.FGBOPAL_ORGN_CODE = FTVORGN.FTVORGN_ORGN_CODE
         and FGBOPAL.FGBOPAL_COAS_CODE = FTVORGN.FTVORGN_COAS_CODE )
   and ( FGBOPAL.FGBOPAL_COAS_CODE = :Main_DD_COA.COA
         and FGBOPAL.FGBOPAL_FSYR_CODE = :Main_DD_FY.FY
         and FGBOPAL.FGBOPAL_FUND_CODE = :Main_DD_Fund.Fund
         and FTVORGN.FTVORGN_EFF_DATE =
         ( select Max(FTVORGN.FTVORGN_EFF_DATE) "MaxDate"
             from FIMSMGR.FTVORGN FTVORGN1 ) )
 group by FGBOPAL.FGBOPAL_ORGN_CODE,
          FTVORGN.FTVORGN_TITLE,
          FTVORGN.FTVORGN_ORGN_CODE||' - '||FTVORGN.FTVORGN_TITLE
 order by FGBOPAL.FGBOPAL_ORGN_CODE

The subquery is not actually correlated to the main query.  It is pulling the latest effective date for all orgs.  Try changing the subquery to this:

  

( select Max(FTVORGN1.FTVORGN_EFF_DATE)
  from FIMSMGR.FTVORGN FTVORGN1
  where FTVORGN1.FTVORGN_COAS_CODE = FTVORGN.FTVORGN_COAS_CODE
  and FTVORGN1.FTVORGN_ORGN_CODE = FTVORGN.FTVORGN_ORGN_CODE
)

My personal preference is to look at the effective date and the next change date: 

and SYSDATE BETWEEN FTVORGN.FTVORGN_EFF_DATE AND FTVORGN.FTVORGN_NCHG_DATE

This tends to make the query less complicated.  It also has the advantage of running the report for different dates (if you swap out "SYSDATE").


Hope this helps!
