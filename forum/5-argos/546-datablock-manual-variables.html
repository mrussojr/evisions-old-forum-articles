I have a user request to update a Manual Variable in the DataBlock.  The report created from the DataBlock uses a value in a Manual Variable to calculate a financial value.  The Report Writer would like to be able to update the Manual Variable instead of submitting a request to the DataBlock designer to update.  I am attempting to build a Form on the Dashboard to view and modify the Manual Variable using a button to trigger.  I know you can 'Set a Variable' on a click but that is a hard coded value.  Is there a way to update a Manual Variable without DataBlock permissions?  I am working on creating a Panel that is viewable only to specific security classes.  I would then us an Edit Box to display the value and allow the user to modify.  Then a button to set that modification.  I tried to write a SQL Variable to SET the Manual Variable, as you would in an SQL query, however it does not validate without a 'UPDATE tablename' entry.  I am trying to avoid creating a specific SQL Schema and Table(s) to query and update.  Anything I am missing in the Argos capabilities?

Let's say the initial value is 5 and the user wants to update it to 10.

 

If you want to persist the value of 10 after the datablock is closed, I'd use a custom table (i know you don't want this path, but can't see other way)

If you don't mind the value going back to 5 everytime the datablock opens, then

I would use a SQL Variable "_finVar" and a text box "txtfinVar" with a text of 5 and a button "cmdUpdate"

The _finVar variable would be something like "Select :txtfinVar as Val From dual where :cmdUpdate is not null"

 

and in your report you'll always use _finVar.Val which will have the most current value while the form is open.

 

hope it helps.
