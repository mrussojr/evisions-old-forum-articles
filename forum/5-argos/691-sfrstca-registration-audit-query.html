This seems simple enough, I want to have parameters for Start Date and End Date for a range in the registration audit trail.  In PL/SQL I can say:

 

AND TO_CHAR(TO_DATE(SFRSTCA_RSTS_DATE, 'DD-MON-YYYY'), 'MM/DD/YYYY') >= '08/19/2018' AND TO_CHAR(TO_DATE(SFRSTCA_RSTS_DATE, 'DD-MON-YYYY'), 'MM/DD/YYYY') <= '08/25/2018'

 

In the Argos Report Query (when validating) I am getting an error:

 

(EOCINativeException) [FireDAC][Phys][Ora] ORA-01843: not a valid month

 

How do I have the date parameters recognize the date since SFRSTCA_RSTS_DATE is in the format MM/DD/YYYY HH:MM:SS AM or FM (8/21/2018 7:15:38 PM)?

Hi Lea,

 

SFRSTCA_RSTS_DATE is should already be in a date format; it is not necessary to use TO_DATE on it.  However, if you are using Date Edit boxes, you shouldn't have to do any string conversion at all: 

 where SFRSTCA.SFRSTCA_RSTS_DATE >=:DE_Start
       and SFRSTCA.SFRSTCA_RSTS_DATE <= :DE_End + 1

HTH

To be sure you are getting all records for the entire date range, you'll truncate the date and chop off the time part with:  trunc(sfrstca_rsts_date)

 

argos and oracle dates work fine together in the where clause most of the time, it's when you try and use those comparisons in subqueries where it might get buggy.  Try to do the date comparison without the type casting first, if that doesn't work, you will basically do what you did above, but the other way around:

 

to_date( to_char ( trunc(sfrstca_rsts_date),'MM-DD-YYYY')) = :argos_variable
