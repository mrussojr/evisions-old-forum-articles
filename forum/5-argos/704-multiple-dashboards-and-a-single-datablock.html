Can you have a single Argos Datablock and have multiple dashboards, with different presentations on each dashboard?

 

The software (Argos 6.0 Build 2148) enables the user to add a dashboard to the datablock, after the system created dashboard has been created. However when you edit that dashboard, it does not let you modify the presentation of the dashboard, i.e., the 'Design' button is greyed out and cannot be pushed. The edit dialog box does advise the user to click the Design button  and in brackets says this is not available for dashboards. So if you believe the message this does not appear to be an authorization issue. However why would the user be allowed to create a dashboard, if it just inherits the main dashboards presentation and this cannot be modified?

 

The workaround is to add forms to the main dashboard and provide either tabs or buttons for the user to access these forms. However under certain circumstances it would be nice to have a separate dashboard altogether to present data and information to the end user in a different way or through alternative selection criteria, without using a report object.

 

Any ideas?

If you are looking for a presentation with multiple screens, you would need to create multiple forms and navigation through them. Once that is done, you could use the dashboard object, or just the datablock, and have multiple saved settings for execution. These can be set and selected from the Dashboard Options drop menu in the datablock/dashboard. So if the dashboard can serve multiple departments or colleges, you could have the selection criteria set for each one. Does that make sense? I will see if I have an example to post.
