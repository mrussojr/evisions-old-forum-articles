We solved our issue so thought I would share it in case someone else has the same problem as we did.

 

This was a big problem for us because we use the interactive API and run Argos reports from Banner Self-Service using Banner Security.  We currently don't use the Web viewer or allow individuals to log into Argos because we do not want to maintain security in two places.

 

Here is the solution:

 

1.       Uninstall java

2.       Open IE and go to http://java.com

3.       Click on ‘free  java  download’

4.       Click on ‘Agree and Start Free Download’

5.       Follow prompts until java installed

6.       Now when you visit the argos applet page, it prompts for running the applet.

 

It turns out the regular J2SE java download is different (JRE integration) and doesn’t work with IE11.

We have some users experiencing the same symptom, Argos just spins and doesn't load, but with the .NET-based Evisions Application Launcher. 

 

This is happening when some users click a link in BannerWeb for an API-enabled report.  I submitted a ticket back in April and I was advised this a known bug.  Evisions has an open enhancement report, known internally as 'AR-3487 Prompt Users for Evisions Application Launcher Installation when Interactive API is Executed', asking for a message to be displayed to alert the user they need to download the application launcher.

 

We have been resolving the issue by having the user go directly to the MAPS server, authenticate, try to launch the Argos client, navigate back to Argos launch page and click the link in the message that now appears below the Argos launch button, "If the application does not launch you may need to download & install the Evisions Application Launcher. "       

 

I thought I would share in case anyone else is encountering this issue with the .NET-based Evisions Application Launcher.

Add like to add two cents as we were also intermittently having a similar issue after months with no problems. We traced the issue to the latest versions of popular browsers like Firefox and Chrome - they've apparently dropped support for the Java Plugin entirely. You can read more here: https://www.java.com/en/download/help/firefox_java.xml and here https://support.mozilla.org/en-US/kb/npapi-plugins

 

We can still get Argos to run with IE just fine (even IE 11). In order to get an Argos report to run inside Firefox, I had to download a 32-bit version of FF, install an Extended Support Release extension and then it would run. What a headache if any of your heavy Argos uses aren't particularly computer savvy!
