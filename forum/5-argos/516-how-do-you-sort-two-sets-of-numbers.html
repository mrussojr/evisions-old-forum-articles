We have a datablock that uses both Regular term values and then CE term values.  Our user has asked if the CE Term values can be sorted such that they come out first in descending order, followed by the standard terms, also sorted in descenidng order, so that the display will be like this:

 

820181

820182

820183

 

201810

201820

201830

 

Is there a way, using only SQL, to do something like this?  Not separate data columns, all  data from one column.

Sure!  How do you differentiate between the two types?  If it is the first character, you could do something like this: 

ORDER BY
CASE SUBSTR(STVTERM_CODE, 1, 1)
WHEN '8' THEN 0 
ELSE 1 END,
STVTERM_CODE

This looks at the first character of a term code (stored in the STVTERM_CODE field in my example).  Term codes starting with an "8" come out first followed by all the other ones.  They are then ordered in ascending order. 

 

Another option would be this: 

ORDER BY
SUBSTR(STVTERM_CODE, 1, 1) DESC,
STVTERM_CODE

This example sorts first by the first character in descending order followed by the entire field in ascending order.

 

Hope this helps!

whenever I have to do a special sort, I will create a "SortBy" field that I can use in my 'order by' clause.  I uncheck "visible" when editing the multi-column box column properties. 

Here's a possible example:

 

select 

case when stvterm_code like '8%' then 1

           when stvterm_code like '2%' then 2

            else 0 end "SortBy",

stvterm_code,

stvterm_desc

 

from  saturn.stvterm

 

order by 1
