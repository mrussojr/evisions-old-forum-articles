After checking Deana's suggestions, you might first verify your query is working the way you want it to work.   You could make a report that only has a detail band and move your data from 'Argos Data' into the detail band and see if it looks like you would expect.  If you want your output sorted alphabetically you should sort on last name and not PIDM.     If you sort on PIDM the names will print in a rather random order.  If that looks correct, I would think your report would look like this:

 

Group Header - use PIDM

Detail band that lists the name,  FY 2018 giving,  FY 18 pledge and the FY17 gifts

Group Footer - sum the fields in the detail line  for that PIDM 

Report Footer - totals for all FY 2018 giving, FY 18 pledge and FY17 gifts

 

By definition - 'a sub detail band is used to add sub reports that add additional information for each record in the detail band'.  It doesn't seem like you would need a sub detail band.

Hope this helps.

John
