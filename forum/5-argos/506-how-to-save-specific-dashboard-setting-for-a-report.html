Did you ever get this to work ?   What is the purpose of the checkbox ?  If the purpose is to filter the data or sort the data,  you could remove the checkbox from your screen and remove the checkbox logic from your SQL and you could use the Filter tab on the edit report screen.  Your SQL would return all the data needed by both reports and you would set custom actions (filter,sort) for each report to do whatever it is the checkbox is doing.  I have use this method when I have multiple reports in the same datablock.
