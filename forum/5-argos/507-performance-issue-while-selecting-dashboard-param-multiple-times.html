As a general rule for myself, I usually try to call the variable only once per query, if possible.   

 

Most of my reporting is done for Financial Aid, so we define most of our reports by aid year, but have to routinely deal with term codes for entering students, so for a report that takes a term code drop down, I'll do something like: 

SELECT  count(*)
  FROM SPRIDEN
       INNER JOIN SARADAP
               ON SARADAP_PIDM = SPRIDEN_PIDM
       INNER JOIN STVTERM
               ON STVTERM_CODE = :Main_DD_Term.TermCode
              AND SARADAP_TERM_CODE_ENTRY = STVTERM_CODE
        LEFT OUTER JOIN RORSTAT
                    ON RORSTAT_PIDM = SPRIDEN_PIDM
                   AND RORSTAT_AIDY_CODE = STVTERM_FA_PROC_YR
WHERE SPRIDEN_CHANGE_IND is null

Then, for the rest of the Query, I can continue to reference STVTERM_CODE instead of calling the variable every time.

 How About:

  

SELECT count(*)
  FROM SGBSTDN
       inner join STVTERM START_TERM
               on START_TERM.STVTERM_CODE = :StartTerm
       inner join STVTERM END_TERM
               on END_TERM.STVTERM_CODE = :EndTerm
 WHERE SGBSTDN_TERM_CODE_EFF between START_TERM.STVTERM_CODE and END_TERM.STVTERM_CODE
