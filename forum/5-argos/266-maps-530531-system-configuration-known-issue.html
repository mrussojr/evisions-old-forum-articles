Posted May 30, 2017

We are aware of a problem introduced in MAPS 5.3.0 that can sometimes result in user sessions being shared or prematurely timed out.  This only affects system configurations in which a single point of entry is utilized to MAPS that results in multiple user connections appearing to come from the same IP address. We have resolved the problem and are working to release the fix in the coming week.  In the meantime, we recommend not upgrading to MAPS 5.3.0 or 5.3.1 if your network configuration matches this scenario.  We regret any inconvenience this has caused and thank you for your patience while we address this matter.
