It only includes the student's name, ID, course information (subject, number, section, grade, institution code, etc). The query works just fine until I attempt to put criteria in the "Order By" clause. I cannot seem to find a way (in the visual designer) to add an Order By Student ID (which exists in both Main & Union queries) without causing an error:

 

Server error number: 003000

Even though SPRIDEN_ID is used in both queries.

 

Is order by just not possible in unions? Seems odd that it would not work. Everything I can find online about Unions and Order By simply says the Order By statement must appear at the end of the query. When I view the sql behind the visual designer, the Order By clause is the last line.

In SQL, you cannot name the columns in ORDER BY with a composite statement, you have to use column position.

 

 

In the visual designer you do a calculated field, look at the image attached, keep in mind the position might be different for your query.
