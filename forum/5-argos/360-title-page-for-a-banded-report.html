This solution is a little hokey but it's the only thing I could come up with!

First, based on your margins and page header, you'll need to determine the # of inches your title area needs to be.  So if your margins are 1 inch on top and bottom, and your page header is .25 inch, then you will need title space of 8.75 inches (11 inches minus 2.25 inches for header and margins).  First, build your report without the title page.  Then, lastly, add a Title band.  Go into options, set the height to 8.75, and then you'll have the space to create your labels, expressions, etc.  If you need to edit the rest of the report (which is now falling off the page and into oblivion), then set the Title band height to 1 inch or something.  Then when you are done editing, set the Title band back to 8.75 and save the report.

 

It has to be a Title band so that it only shows up on the first page.  Page header won't work because you don't have the option to only show it on the first page.  Subquery and detail bands are about the only ones that will "grow and shrink" based on the objects contained therein.  My test Title band wouldn't grow even if my rtf or expression field was 9 inches tall.
