You could try a NVL in the query itself.  Using Registration date, I could pull:

 

SELECT *

  FROM SFRSTCR

 WHERE SFRSTCR_RSTS_DATE BETWEEN NVL(:DateEdit1, '01-JAN-1900') and SYSDATE
