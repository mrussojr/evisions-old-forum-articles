Personally, I'd ask the user to determine beforehand what the largest value is from the list of values they want to pass to the query.  It would make datablock development faster and easier to maintain.  That being said, I understand that sometimes we just do what we're told to do.

   

For example, if the user wants to pass to the <= operator the values of "5,7,6,12,33,14,91" a quick assessment would reveal that the largest value is "91".  The user would then pass only that value to the <= operator.  That is because all the other values the user wants to pass to the <= operator are also <= 91.  You wouldn't have to modify your existing query and the user gets the records they want from the query.

 

Identifying the largest value in a list could be more challenging if
A) the list of values is rather large
B) the list of values aren't strictly numeric

 

I made some example datablocks that determine the largest value supplied by the user.  Point them to any of your Oracle database connections; no data is actually queried from Banner.  One is for character/numeric datatypes and the other is for date datatype.  The links for them are below:

Right, what I provided answers your original question of, "How do I use multiple criteria with a <=". 

 

What you just described is a different problem to solve.  And yes, the solution is to change the operator from '<=' to '='.  If you pass multiple parameters to the '=' operator Argos will dynamically change the operator to 'IN'.

 

"I want records where x <= (1 or 2 or 3 or 5 or 8)"  is a different stated objective compared to, "I want records where x = (1 or 2 or 3 or 5 or 8)".
