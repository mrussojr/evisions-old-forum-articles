One way to achieve this is to leverage SQL variables (you'll need "DML" access at the ADO connection level though, so talk to your Argos Admin).  Create a SQL variable that executes the procedure you want to call between a begin and end keyword. Now, this runs the second you test/validate the variable or open the datablock (which we don't want!) so we'll need to add a condition we can manipulate with the button click:

 

BEGIN

IF :wait4btn = 'Y' THEN

   PROCEDURE_NAME(any, passed, values);

END IF;

END;

 

(Incidentally, this will keep the procedure from firing before you are ready when you test it as well!)

 

:wait4btn will be a variable we create. Open your datablock, on the variables tab, click the green plus and add a manual variable. Name it wait4btn, leave the default type as string and type N in the value box. it's very important to ensure there are NO white space characters after it (this happens to me often).

 

Now add a button to your form. Use the on click property to add three actions, in this exact order:

  1. Set Variable - variable name wait4btn; set it to Y (watch those spaces/hard returns!)
  2. Refresh variable - the sql variable that executes the procedure you want to call
  3. Set Variable - variable name wait4btn: set it back to N so the procedure doesn't keep executing.
