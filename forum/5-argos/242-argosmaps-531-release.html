Posted May 9, 2017

Evisions is pleased to announce the release of Argos and MAPS version 5.3.1, which resolve two issues from the 5.3.0 release:
 

  • When editing a schedule's existing "Execute the report" task, any previously configured parameter values were cleared from the editor, requiring you to enter them again in order to make changes.
  • Attempting to load an Argos DataBlock that was using a data connection set to "use connection username and password" with the credentials left blank resulted in the error "We were unable to load your DataBlock.  Please contact your Administrator if this problem persists." when accessing it in the Web Viewer, instead of prompting the user for credentials as it should have.

 

This release includes updates to the Argos client and the MAPS service.

 

As always, we highly recommend installing updates in a test environment before applying them to your production environment.  You should make sure that a current backup is available in case of any unforeseen issues.

 

For installation instructions, please refer to the Argos 5.3.1 Release Guide. and MAPS 5.3.1 Release Guide.

 

If you have any difficulty with the updates, just contact our HelpDesk and we will be glad to assist you.
