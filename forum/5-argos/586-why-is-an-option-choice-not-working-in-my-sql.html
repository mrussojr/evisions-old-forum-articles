Okay.  If I understand correctly, you want the user to be able to select multiple terms.  If the checkbox is checked, then the query only returns the selected term(s).  If the checkbox is unchecked, then the query should return all the terms below the greatest selected term.  Because of how Argos processes queries, using less than or greater than can be a challenge.

 

If that is what you're trying to do, I have an ugly solution:

 

Create SqlVariable1, which will list all the selected terms in the list box: 

select STVTERM.STVTERM_CODE as "TermCode"
  from SATURN.STVTERM STVTERM
 where STVTERM.STVTERM_CODE = :Parm_LB_Term.TermCode

Create SqlVariable2, which will list the greatest selected term: 

select Max( STVTERM.STVTERM_CODE ) as "TermCode"
  from SATURN.STVTERM STVTERM
 where STVTERM.STVTERM_CODE = :SqlVariable1.TermCode

Create SqlVariable3, which will list all the applicable terms: 

select STVTERM.STVTERM_CODE as "TermCode"
  from SATURN.STVTERM STVTERM
 where ( STVTERM.STVTERM_CODE = :SqlVariable1.TermCode )
       or ( :Parm_Check_term = 0 and STVTERM.STVTERM_CODE < :SqlVariable2.TermCode )

Finally, in your  multi-column query, you can select records where the term code equals SqlVariable3.TermCode.  Alternatively, your multi-column query could incorporate the query from SqlVariable3.  (I also recommend renaming the variables for something more understandable if you need to make changes later.)

 

HTH
