We have our admission letters set up to print from Argos and then scheduled to move a pdf copy to BDM. We had to set up each letter type through GURMAIL in Banner. We also had to have a communication plan for the letters and an update to GURMAIL so students wouldn't get more than one copy of their letter.

I created a Datablock for everything, then reports were filtered by letter type and other criteria to get the right pieces. We have admit letters which include the actual admit letter, a notification letter with their ID, a letter from the president, etc. Then we also have deny, waitlist, etc other types of letters.

 

I use banded reports with dynamic paragraphs based on codes in Banner to know which type of paragraph to put in the letter based on the codes.

 

The letters are printed daily before any admissions decisions are entered for the day. Once the letters are done, the processors start making decisions again. This ensures we don't miss someone.
