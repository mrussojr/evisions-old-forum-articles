You cannot have 2 active database connections and pull data from both into one query -- say into a single Multicolumn Listbox. To do something like that you would need the 2 databases dblink'd at the database level. That's done in Oracle and has nothing to do with Argos. If the 2 are dblink'd correctly, then you can use one connection to pull data from both. 

 

But, you can have 2 separate queries from 2 sources on the same dashboard--one database in Multicolumn Listbox 1 and the other database in Multicolumn Listbox 2. For example, I have access to Banner databases from 2 different schools so I could develop a dashboard that lists all the STVMAJR entries from each in side-by-side Multicolumn Listboxes. You do that by using the "Connection" link at the bottom of the free-type SQL query screen (under the abc button). Normally it says "Connection: default", but you can click on it and change it to a different db connection for one of the listboxes.

 

I'm not sure how to do it in banded reports (we almost never use them), but I expect it would be similar with each source having to go into a different band.

I agree with Lynne.  

 

If you did need to connect to more than one database in a banded report, without the use of a dblink, you could use the Dataset feature.  From the Banded Report Editor, click 'Show/Hide Dataset', click the '+' sign to add a Dataset, click '<default>' next to 'Connection:' and select the database you want to query instead of the default connection associated with the Datablock, create your query but make sure you link it with the driving query.  One caution on Datasets -- they are noticeably slower to process.
