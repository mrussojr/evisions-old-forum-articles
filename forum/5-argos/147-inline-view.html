I am hoping that someone may have a resolve for this, currently we run a lot of Brio's and MS Access where we need a query to be based on the results of a previous query, now I know this can be done as an inline view if you Manually type the entire sql.   Has anyone figured out, or is there a way to perform those inline views with the gui.   Though I may be very good at sql not all of our data block designers have mastered that yet.

 

here is a Banner example

select distinct
glvextr_key as PIDM,
argosuser.f_gcc_get_has_payment_plan(a.GLVEXTR_KEY,'201602') as "PaymentPlan",
f_gcc_format_name(glvextr_key,'L30') as Last_name,
f_gcc_format_name(glvextr_key,'F30') as First_name,
f_gcc_get_spridenid(glvextr_key) as ID,
tbraccd_amount,
tbraccd_entry_date,
(select sum(tbraccd_balance)
       from tbraccd d
       where d.tbraccd_pidm = a.glvextr_key
          and d.tbraccd_entry_date >= :frmMain_mc_term.STVTERM_START_DATE
          group by d.tbraccd_pidm) as Balance
 from glvextr a,
   (select * from tbraccd t
     where tbraccd_pidm in (select b.glvextr_key from glvextr b
                                where b.GLVEXTR_APPLICATION = 'BUSINESS_OFFICE'
                                and b.GLVEXTR_SELECTION = 'PURGE_SELECTION'
                                and b.GLVEXTR_USER_ID = 'KRISTIN_YUNKER'
                                and t.tbraccd_pidm = b.glvextr_key)
      and tbraccd_detail_code like 'P%'
      and tbraccd_entry_date >= :frmMain_mc_term.STVTERM_START_DATE )
   where a.GLVEXTR_APPLICATION = 'BUSINESS_OFFICE'
   and a.GLVEXTR_SELECTION = 'PURGE_SELECTION'
   and a.GLVEXTR_USER_ID = 'TAMMY_WALDEN'
   and a.glvextr_key = tbraccd_pidm(+)
order by 3
 

 

 

Any recommendations would be great.
