There are a couple of known issues in our current release of MAPS/Argos 5.3.  Please see below for details.  Our engineering team has resolved the issues and are currently in the QA process.  We should have a hotfix shortly. If you have any questions, please create a HelpDesk ticket so we can further assist you. 

 

Issue Number 1: AR-4921

Users are no longer prompted for their database credentials in Argos Web Viewer.   The following error is displayed:

 

Error Code: -1

We were unable to load your DataBlock.  Please contact your Administrator if this problem persists.

 

The issue occurs when the Data Connection is set to use the connection username and password and the username and password is blank in the Data Link Properties.  Prior to Argos version 5.3, if the Data Connection is set up this way, the user should be prompted for their database credentials.  The work around on version 5.3 is to launch the Argos client and run the Dashboard/Report from the Argos client which will continue to prompt the user to enter database credentials.  For more information, please see the following Knowledge Base article:

 

Error Code: -1 User not prompted for correct ADO Credentials

 

Issue Number 2: AR-4867

Parameter values are undefined every time a user goes to edit the “Execute the Report” task of a schedule. 

 

If the schedule's parameter values were configured in the past and they do not need to be changed, the schedule will continue to run using those values. Users should not click the "OK" button on the "Edit Execute Report" window, but close the window. Clicking the OK button while the parameter values are undefined will cause Argos to clear the parameter values previously set to the task.

If the user needs to make changes to the schedule's parameter values, he/she can set the new values to the parameters and click the OK button. This will save the new parameter values for the next schedule run.  For more information, please see the following Knowledge Base article:

 

Parameter objects contain undefined values when editing the "Execute the Report" task in Argos 5.3
