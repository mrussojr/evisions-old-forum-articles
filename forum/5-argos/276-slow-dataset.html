It sounds to me like you might just be dealing with a lot of rows. Limiting down a query to FONDS='10' for example will give you less rows, allowing your query to run faster. Are you creating a new dataset for every summary band? If the report is running 10+ queries with a lot of rows, it could be slow. Maybe posting an example of the queries can allow us to actually see what is slowing it down.

 

ArgosData is just the name for the report query in the datablock. Banded reports can also have independent datasets inside them to allow for more customization.
