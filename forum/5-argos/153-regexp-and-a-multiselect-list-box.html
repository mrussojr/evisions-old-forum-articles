To answer your last question, I believe that when a query receives a multi-selected parameter Argos dynamically changes any '=' operators to an 'IN' operator and parses the array of values into (value1,value2,value3,etc).  I think the use of regular expressions and common table expressions aren't necessary.  A conditional WHERE clause will usually suffice.
