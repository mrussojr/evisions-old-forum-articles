Hi Vijaya,

I have received that error when my datablock receives a lot of data from the server--more data than my workstation can handle.  Watching the Argos application, I see that the memory increases based on the number of fields needing to be rendered.  

 

Can you reduce the amount of data being returned?  E.g., can you limit the number of rows or columns?  Can your query consolidate columns?  For example, for financial records, I typically combine the Fiscal Year and Fiscal Period into a single column  (e.g., 2018-12).  Also, unless the users need to see the data on screen in a multi-column list, I prefer returning the details in a report query.
