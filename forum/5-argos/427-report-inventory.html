Benjamin,

 

Not sure if this will help you or not, but what I've done for our campus uses the actual Argos database to report against.  In this instance I've created a DataBlock that uses the internal Argos tables to create a way to search for reports, but it may help you create an inventory.

 

You'll need to follow the guide for reporting against Argos to get it up and running, but it wasn't too complicated, and the eVisions guide is very easy to follow to get you there. 

 

FYI, you'll need MAPS admin rights to set up the datasource, as well as admin rights for your server to install the sqlite driver if it isn't already installed.

I don't try to manually keep a list of datablocks/reports. Instead I set up an ADO connection to the MAPS Data Warehouse (like Michael mentioned above). Then I could create a datablock that lists datablock and/or reports. 

 

I use this to get the datablocks:

select DataBlock.Name as "Datablock",
       DataBlock.Description,
       Connection.Name as "Connection",
       DataBlock.Path,
       Folder.Name as "Folder",
       DataBlock.LastRunDate,
       DataBlock.Author,
       DataBlock.CreateDate,
       DataBlock.ModifiedBy,
       DataBlock.ModifyDate,
       DataBlock.Id,
       LENGTH(TRIM(DataBlock.Description)) as "LengthDesc"
  from ( DataBlock left join Connection on DataBlock.Connection_Id = Connection.Id ) left join Folder on DataBlock.Folder_Id = Folder.Id
 where :main_BT_GetDatablocks IS NOT NULL
 order by DataBlock.Path,
          DataBlock.Name
 

And this to get the reports that belong to a particular datablock:

select  CASE
          WHEN TRIM(Report.Name) = '@DEFAULT@'
            THEN 'Default Dashboard'
          ELSE TRIM(Report.Name)
        END as "ReportName",
       CASE
         WHEN TRIM(Report.Description) = 'System Object (limited options)'
           THEN 'None given--see datablock description'
         ELSE TRIM(Report.Description)
       END as "ReportDescription",
       Report.ModifyDate as "ReportDate"
  from Report
 where Report.DataBlock_Id = :main_MC_DatablockList.Id
 order by Report.DataBlock_Id,
          Report.Name          
 

Hope that helps you with starting your inventory.  -- Lynne
