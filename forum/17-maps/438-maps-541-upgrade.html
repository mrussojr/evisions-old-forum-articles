We upgraded our development server last Friday 10/27 so we have been on 5.4.1 for only 4 full days.  No problems have been reported and services have been stable since the upgrade.  Our developments server experiences more daily use than our production server so it receiving some decent stress testing.

 

Here are some details from the diagnostics files I generated before and after the upgrade.  One caution, there have been cases where the diagnostic file is out of sync with reality.  I notified Evisions about this issue and they created AR-5156: MAPS diagnostics shows incorrect version for AWS.

 

Server:

Windows Server 2012 R2 (Version 6.3, Build 0, 64 bit)

 

Before upgrade:

MAP service - 5.2.2.1436

Argos app - 5.2.1.1593

Config app - 5.2.1.741

Argos report runner app - 5.2.1.854

Argos Web viewer app - 5.2.1.1155 [really 5.2.0.2 when you click 'Help', 'About' from within AWV]

Argos mapplet - 5.2.1.1155

 

After upgrade: 

MAP service - 5.4.1.1662

Argos app - 5.4.1.1986

Config app - 5.4.1.925

Argos report runner app - 5.4.0.1049

Argos Web viewer app - 5.4.0.1529 [really 5.4.0.8 when you click 'Help', 'About' from within AWV]

Argos mapplet - 5.4.0.1529
