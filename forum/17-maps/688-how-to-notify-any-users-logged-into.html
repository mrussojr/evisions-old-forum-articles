Support documentation includes the phrase "Notify any users logged into the MAPS Applications (IntelleCheck, Argos, FormFusion and DataMasque) to save their work and log off so there is no unintentional loss of work" when it is necessary to make changes to the MAPS service.  As a new administrator for MAPS and Argos about to run my first production upgrade, I'm looking for ideas on how do you do that?  How do I know who is logged into the MAPS applications at that moment?  Is there a way within MAPS to send notifications to users?  Thanks for your thoughts!

Hi Diana,

 

Using the MAP Server Configuration tool, navigate to the Inspector tab .  From there, you can see the logged-in users as well as broadcast a message.  

 

I also recommend setting a Server Message that users can view when they log in.  We usually do this well in advance of our upgrade so that users know when the system will be down.

 

HTH
