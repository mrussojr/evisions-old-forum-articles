For a number of our connections, we use a different database role depending on the group the user is in.  For example: 

SET ROLE *username* IDENTIFIED BY *password*

We have groups for financial aid, HR, payroll, etc. that each allow the group viewing rights to "their" data.  The caveat is that users cannot be in multiple groups of this type as you don't know which group/rule Argos will apply.

 

Hope that helps!
