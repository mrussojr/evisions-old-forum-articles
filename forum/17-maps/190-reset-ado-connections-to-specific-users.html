As far as I can tell, there is no point-and-click way to do that with the MAPS Config tool.  MAPS Config will grant access to Applications to use Connection Pools or individual Connections.  It does not grant access to Groups or Users to use Connection Pools or individual Connections.

 

You could accomplish it by using a Rule Script on individual Connections.  For example, we have some Rule Scripts active on individual Connections that exclude work-studies from using Argos outside of business hours.  Our DBA created the scripts since they lookup Roles, grant privileges, etc.  The settings can be found by double-clicking an individual Connection, then selecting 'User/Group Rules'.

I concur with Todd; I haven't found a way to restrict at the Connection Pool level.  What we do primarily is to allow/block access to various ADO Connections by group membership.  That way, if they are not a member of the group, they cannot access/see the connection.  (We also use rule scripts, based on group membership, to set data access permissions.)
