Posted November 30, 2017

Evisions is pleased to announce the release of FormFusion version 3.6, which includes the following changes:

  • Resolved an issue where running load balancing software or security vulnerability scanning software could cause FormFusion to become unstable.
  • Resolved an issue where UTF-8 characters in the .lis file were sometimes causing jobs to fail.
  • UTF-8 characters used in the body of emails sent from FormFusion will now display correctly.
  • Made several minor user interface enhancements to increase product accessibility and usability.
  • Additional enhancements and resolved issues.
     

This release includes updates to the FormFusion client and FormFusion mapplet.  FormFusion 3.6 requires that you be on MAPS version 5.2 or higher.  If you have not already upgraded to MAPS version 5.2 or higher, please refer to the MAPS release documentation.

 

As always, we highly recommend installing updates in a test environment before applying them to your production environment. You should make sure that a current backup is available in case of any unforeseen issues.
 
For installation instructions, please refer to the FormFusion 3.6 Release Guide.
 
If you have any difficulty with the updates, just contact our HelpDesk, as we’re here to help. We would like to thank the FormFusion community for all the support, suggestions and help in making this new version a reality. We look forward to hearing your feedback, and encourage everyone to keep the great ideas coming!
