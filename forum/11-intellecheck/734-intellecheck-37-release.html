Posted January 10

Evisions is pleased to announce the release of IntelleCheck version 3.7, which includes the following changes:

  • IntelleCheck now supports check numbers up to 16 digits long.
  • Check numbers that contain an "@" symbol now appear when searching on the check number.
  • Text within IntelleCheck now scales based on the Windows DPI setting.
  • Made several minor accessibility improvements relating to hyperlinks.
  • Updated the system requirements to clarify that printer drivers used with IntelleCheck must support PCL 5.
     

This release includes updates to the IntelleCheck client and mapplet. There are no updates to the IntelleCheck database.  Some of the changes require that you be running the latest version of the MAPS service (currently at 6.1.1.2018).  If you need to upgrade MAPS, please refer to the MAPS release documentation.

 

As always, we highly recommend installing updates in a test environment before applying them to your production environment. You should make sure that a current backup is available in case of any unforeseen issues.
 
For installation instructions, please refer to the IntelleCheck 3.7 Release Guide.
 
If you have any difficulty with the updates, just contact our HelpDesk, as we’re here to help. We would like to thank the IntelleCheck community for all the support, suggestions and help in making this new version a reality. We look forward to hearing your feedback, and encourage everyone to keep the great ideas coming!
