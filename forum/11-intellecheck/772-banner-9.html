In Banner 9, all administrative users need an Oracle (Banner) account.  The account must have the appropriate security to run any jobs, access forms, etc....just like Banner 8.  However, the user credentials are managed using LDAP, EIS, etc., so the user does not need to know their Oracle password.  The Oracle password can be expired, as long as the Oracle account is not locked. IntelleCheck follows this pattern. 

 

Logging into IntelleCheck depends on how you authenticate to MAPS. 

 

Any users that connect directly to the Banner database (via TOAD for example) will need need to keep their Oracle accounts active (not expired).  There is a Banner form that can be used to reset passwords (not sure of the form name, but entering password in the search box on appnav will get you there).  In addition, Ellucian has provided a Credential Management Utility that can be used without logging into Banner. 

 

Hope this helps,
