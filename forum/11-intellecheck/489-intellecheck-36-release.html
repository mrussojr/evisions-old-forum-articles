Posted January 18, 2018

Evisions is pleased to announce the release of IntelleCheck version 3.6, which includes the following changes:

  • Increased the length of the USERID fields in the EVIBCCHK and EVIBCAUD tables to 60 characters.
  • When imaging payments, you now have the option to generate a single PDF for all payments.
  • Added an option to the positive pay configuration to trim whitespace from the ends of fields.
  • The Check Register Report with Invoice Detail can now filter the report based on VPDI code.
  • Additional enhancements and resolved issues.
     

This release includes updates to the IntelleCheck client, mapplet, and database. There are no corresponding updates to MAPS; however, you should ensure that you are running the latest version of MAPS (currently at 5.4.1) since this is the version that was tested with IntelleCheck 3.6.  If you need to upgrade MAPS, please refer to the MAPS release documentation.

 

As always, we highly recommend installing updates in a test environment before applying them to your production environment. You should make sure that a current backup is available in case of any unforeseen issues.
 
For installation instructions, please refer to the IntelleCheck 3.6 Release Guide.
 
If you have any difficulty with the updates, just contact our HelpDesk, as we’re here to help. We would like to thank the IntelleCheck community for all the support, suggestions and help in making this new version a reality. We look forward to hearing your feedback, and encourage everyone to keep the great ideas coming!
