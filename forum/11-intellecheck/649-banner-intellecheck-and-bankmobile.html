We started the first running of BankMobile through Intellecheck coming from our Banner ERP.  They did a great job of documenting the process from the check printing process and FABCHKP through Intellecheck, but we were told to run the check printing process the first time and it will get just the BankMobile students.  They will be processed through Intellecheck and an "H" added as the first character of the check number.  We would then run the second time for the non-BankMobile students with old check numbers from the same bank.  The first run, we got 2 students that were not BankMobile and instead of the H0000001 and H0000002, we got 00000001 and 00000002. 

 

So, there must be something folks are doing during or after the refund process that assures a certain batch of refunds is BankMobile and goes to one bank with check numbers with an "H" in the first position and the other batch of non-BankMobile refunds that will go to a different bank with current check numbers.  

 

Is there anyone out there who has implemented the BankMobile process using Intellecheck and has Banner as their ERP.  I would like to see how others are handling this separation of batches.

I was given a document "BankMobile Disbursements  File Specifications - IntelleCheck   Version 25" which had instructions on setting it up.  You basically exchange server IP addresses with the BankMobile project manager.  If you're not in the networking area, you will need a network administrator who can do the actual work on  the server.  The final 5 pages of the document gives all the specifics and it would be coordinated through the project manager at BankMobile.  They should supply you with the document and were very helpful in answering any questions we had.
